# UX-CHANGE-04 · 城镇枢纽 + 五子板块布局重设计(按用户参考图)

> 作用域:本文把 UX-CHANGE-02 立起的「城镇枢纽 + 四覆盖式子板块」从**抽象占位**(入口钮 +
> `.visible` overlay)细化为**每板块的目标布局与信息架构(IA)**,依据 = 用户存入
> `harness/feedbacks/refDesigns/` 的 6 张参考图。**只定"每个区放什么、怎么导航、有哪些交互态",
> 不画像素**(配色/图标/建筑美术 = Art Spec)。参考图信息量大(每日任务/活动/成就/邮件/货币…),
> **多数 test-2 无对应系统 —— 本文逐板块标注"映射现有 / 仅参考(略或缓)",绝不为没影系统提前抽象。**

## 1. 触发 / Trigger
UX-CHANGE-02 把城镇定为「枢纽 + 四子板块(工匠/小队/酒馆/出征)」并已落码,但子板块布局是**最小占位**
(三栏堆控件、无统一 IA)。用户产出 6 张参考图(城镇/小队/打造/出征/酒馆/背包),给出每个板块的
**目标大体布局**,要求过一遍重设计。恰逢 BACKLOG Next① **全局 UI·juice 轮**在即——本文正是那一轮的
**布局蓝图**(避免上皮时各板块各做各的、漏空/错/确认态)。

## 2. 现状诊断 / Diagnosis(根因)
**根因:城镇五板块从未有过一份"信息架构(哪几区、各区放什么、区间怎么导航)"。** 当前
`town_view.gd` 把每板块拍成线性控件流(选槽→背包→按钮),没有"主对象区 / 候选区 / 详情区 / 操作区"
的分区心智,于是:
- **小队**:装备槽 + 背包平铺,无"当前成员→其装备→其属性"的视觉主从(EI-F1 / UX-CHANGE-03 已诊断的成员轴缺失,是其中一例)。
- **打造**:仅"选槽 + 强化 +1"一条线,无"选哪件→看升级预览→看属性增量→确认"的分区。
- **出征**:仅关卡钮列表,**不预览该关敌人/掉落**,玩家选关全凭记忆。
- **背包**:探索内只读掉落预览尚可,但城镇缺一个**全物品管理板**(分类/筛选/详情/分解)。
- **城镇枢纽**:四入口是抽象钮,不是"走进某座建筑"的空间枢纽,陪伴感/场所感弱。
参考图的价值 = 给每板块一套**经过验证的三区(或左中右)IA**,正好补这层。

## 3. 目标形态 / Target layout(逐板块 · delta vs UX-MAP §3)

> **窗口几何前提(已裁决·2026-06-20·SMM,见 §6 风险①)**:城镇**复用 M3.EXPANDED 贴底全宽窗(800×250)**,
> 不新增几何态——现状 M2.TOWN 已渲染于 EXPANDED,且本套参考图为宽幅 ~3.2:1 正是贴底窗画幅。**故以下各板块
> A 层布局一律按 800×250 贴底窗排版**;"城镇专属放大窗"= 条件触发未来债(仅当 UI 轮实证 250px 装不下),
> 详见 STATE-MACHINES.md §6 #8。

### 3.0 城镇枢纽 Town Hub(城镇.png)
**目标 IA = 一幕"小镇场景" + 四周边 HUD;四座建筑 = 四个空间入口(取代抽象入口钮)。**
| 区位 | 放什么 | 映射 |
|---|---|---|
| 中央场景 | **4 座建筑 = 入口**:打造 / 酒馆 / 小队 / 出征,各 = 建筑 + 名牌 + 一行副标 | ✅ 映射现四子板块(仅把"钮"升为"建筑入口") |
| 顶左 | 「城镇」标题牌 | ✅ |
| 顶右 HUD | `[☰]` 主菜单 + ⚙ 设置 | ✅ 已存在 |
| 顶右·货币条 | (参考有金币/钻石×3) | ⚠ **仅参考·略**:test-2 无货币经济,只有分解材料 → 不放货币条;如需可放"材料总览"小角标(flag GD) |
| 底左 | 角色头像 + 名 + Lv(+ **战力**速览) | ◐ 头像/名/Lv 映射 roster(v1 单人);**战力 = 派生新信息位**(flag GD,见 §6 ③) |
| 底中·快捷行 | **背包**快捷入口 | ✅ 背包映射(每日任务/活动/成就/邮件 = ⚠仅参考·略,无系统) |
| 底右·收益 | (参考有 收益/秒 + 领取) | ⚠ **仅参考·略**:test-2 挂机产出 = 掉落(探索时自动拾取),城镇是**暂停态**无"秒收益";保留现「进度速览(第 N 关)」即可 |

**交互态**:`default`(四建筑可点亮/悬停高亮)。进任一建筑 = 打开对应子板块 overlay。

### 3.1 小队 Party(小队.png)
**目标 IA = 左成员栏 / 中主成员(头像+装备槽) / 右属性·技能 / 底装备候选。**
| 区位 | 放什么 | 映射 |
|---|---|---|
| 左·成员栏 | **成员列表**(头像+名+Lv,选中高亮)+「添加成员」 | ◐ **= UX-CHANGE-03 成员轴的最终形态**:由"顶部 tab-row"**改为"左列表"**(更宽裕、与参考一致)。v1 单人=单条目不可切(interim 标签退化态);「添加成员」=招募 v2 占位/disabled |
| 中·主成员 | 选中成员**大头像** + 环绕的**装备槽** + 战力 | ◐ 槽位 = test-2 **3 槽**(weapon/armor/accessory),非参考的 6 槽;战力 = 派生(flag GD) |
| 右·属性详情 | **8 维属性** + 「对比」开关(换装时绿↑红↓增量) | ✅ 映射(对比逻辑已存在) |
| 右·技能 | (参考有技能行) | ⚠ **仅参考·略/只读**:skill_component 是战斗用,城镇无技能管理;v1 至多只读图标,或略(flag GD) |
| 底·装备候选 | 类型筛选 tab + **背包中可换件网格** + 操作(换/卸下/一键装备) | ✅ 换装映射 equip_from_bag/unequip_to_bag;**一键装备 = 便利新功能**(flag);强化主入口归"打造",此处至多深链 |

**交互态**:`单成员`(左栏单条不可切)/ `多成员`(可切·🔜招募 v2)/ 选中槽`空槽` / `(无可换件)` / 换装`对比增量`。

### 3.2 打造 Forge(打造.png)
**目标 IA = 顶三 tab(分解/强化/制作) / 左材料背包 / 中升级预览 / 右属性增量预览 / 底操作。**
| 区位 | 放什么 | 映射 |
|---|---|---|
| 顶·tab | **强化** / 分解 / 制作 | ◐ **强化 = 映射**(enhance_item);**分解 = 管线已存在**(产材料),可在此或背包做(见 §3.5 收口);**制作 = ⚠仅参考·缓**:无 crafting 系统,占位 tab(flag GD/Producer) |
| 左·材料背包 | 材料网格(按 部位×稀有度)+ 筛选 | ✅ 映射 materials(get_material) |
| 中·强化预览 | 选中件 Lv.N→Lv.N+1 + 星级 + 材料消耗(持有/需要) | ◐ 映射(cost_for_level / 满级 / 材料不足);**「成功率 75%」= ⚠不适用**:test-2 强化是**确定性 +1 无 RNG**(player_state `enhance_item` 守 i7)→ **不放成功率、不引入失败概率**(硬 NO:不造没影系统) |
| 右·增量预览 | 强化后 8 维增量(攻击 120→138 +x%) | ✅ 可派生 |
| 底·操作 | 强化 +1 / 一键强化 / 返回 | ◐ 强化映射;**一键强化(连强到满或材料耗尽)= 便利新功能**(flag) |

**交互态**:`可强化` / `满级` / `材料不足`(按钮 disabled) / `分解选件` / `制作·敬请期待`(占位)。

### 3.3 出征 Depart(出征.png)
**目标 IA = 左关卡列表 / 中关卡预览(场景+敌人+掉落) / 右我方小队·战力对比 / 底出征操作。**
| 区位 | 放什么 | 映射 |
|---|---|---|
| 左·关卡列表 | 关卡(已通关✓ / 当前 / 未解锁锁)+ 进度 | ✅ 映射 stages / max_unlocked_stage(未解锁置灰) |
| 中·关卡预览 | 关卡场景图 + **敌人预览** + **可能掉落预览** | ◐ **新信息位·数据已有**:敌人(enemy_def)、掉落表可预览 → 交 Planner(把数据接到预览区);场景图 = Art Spec |
| 右·我方 | 我的小队 + **战力对比 / 推荐战力** | ◐ 战力 = 派生(flag GD);"推荐战力"需关卡难度基线(flag GD/数据) |
| 右·自动战斗开关 | (参考有 自动战斗 toggle) | ⚠ **仅参考·略**:test-2 **恒自动战斗**(支柱 1),开关冗余 → 不放 |
| 底·操作 | 返回城镇 / 开始出征 / 继续挂机 | ✅ 映射:出击=on_depart(i,0)、继续挂机=on_depart(-1) |

**交互态**:选中关`已解锁`(出击可点) / `未解锁`(锁·出击灰) / `当前进度`(继续挂机高亮)。

### 3.4 酒馆 Tavern(酒馆.png)—— 升级版占位(招募 = v2)
参考给的是一个**更丰满的占位**:酒保场景 + 3 个锁定招募位(战士/弓手/法师)+「功能预告」文案 +
招募钮(disabled)。**test-2 酒馆本就 scope = 占位**(UX-CHANGE-02 / BACKLOG Later)→ 本文仅把
"敬请期待"一行**升级为该占位布局**(锁定招募位 + 功能预告),低优先、无逻辑。**招募实装 = 招募 v2**
(那时配 UX-CHANGE-03 成员轴蓝图)。**交互态**:`占位`(招募钮 disabled / 返回可用)。

### 3.5 背包 Inventory(背包.png)—— 城镇全物品管理板(区别于探索只读掉落预览)
**目标 IA = 顶分类 tab / 左筛选(稀有度+类型) / 中物品网格 / 右选中详情 / 底整理·分解。**
| 区位 | 放什么 | 映射 |
|---|---|---|
| 顶·tab | 全部 / 装备 / 材料 (/消耗·任务) | ◐ 全部·装备·材料映射;**消耗/任务 = ⚠仅参考·略**(无消耗品/任务物品) |
| 左·筛选 | 稀有度(白/蓝/金)+ 类型(武器/防具/饰品) | ✅ 映射 3 稀有度 + 3 槽类型 |
| 中·网格 | 物品网格(N/容量) | ✅ 映射 bag |
| 右·详情 | 选中件 名/稀有度·类型/属性/描述 | ✅ 映射 ItemInstance |
| 底·操作 | 整理 / 批量选择 / **分解** / 返回 | ◐ **分解 = 管线已存在**(产材料)→ 背包是分解主入口;整理/批量选择 = 便利新功能(flag) |

> **背包 vs 小队底部候选区的分工**:小队底部 = **快速换装条**(仅可装备件、为当前成员);背包板 =
> **全物品管理**(含材料、分解、详情)。二者互补、不重复。**探索内**仍保留**极简只读掉落预览**(支柱 1,不变)。

## 4. 调整策略 / Strategy(分两层,按系统支撑切)
1. **A 层(映射现有系统,= 全局 UI·juice 轮的布局蓝图)**:把上述各板块**有 ✅/◐ 现有数据支撑的区**
   按参考 IA 重排——城镇建筑入口、小队(成员栏+主成员+对比+候选)、打造·强化 tab(无成功率)、
   出征(关卡列表 + 敌人/掉落预览 + 出击/继续)、背包板(装备/材料 tab+筛选+详情+分解)、酒馆占位升级。
   **这是本文主交付,直接喂 Art Spec 那一轮**;其中"出征敌人/掉落预览""战力派生"需 Planner 接数据。
2. **B 层(无系统支撑,deferred,仅标注不建)**:货币条 / 收益秒 / 每日任务·活动·成就·邮件 / 制作 /
   技能管理 / 消耗·任务物品 / 多成员招募 / 自动战斗开关 / 成功率——**均 ⚠仅参考**,等各自系统真立项再做。

> **升级连续性**:A 层布局以"区位"为骨架,B 层项都对应某个**预留空区**(如顶右货币位、小队右技能位、
> 打造制作 tab)——将来系统落地**填进预留区即可,无返工**。

## 5. 影响面与迁移 / Blast radius
- **屏/overlay**:仅城镇枢纽 + 五子板块(小队/打造/出征/酒馆/背包)**内部重排**;**不新增顶层屏、
  不动 §2 顶层导航图、不改 Flow FSM**(同 UX-CHANGE-02/03:子板块 = overlay,非 Flow 态)。
- **窗口几何 → 【SMM 已裁决:无结构性影响】**:城镇**复用 M3.EXPANDED 贴底窗,不改 M3、不动 MainArea 结构**
  (见 §6 ① + STATE-MACHINES.md §6 #8)。探索极小窗不变。"放大窗"= 条件触发未来债,本期不触发。
- **新信息位(需 Planner 接数据,非新系统)**:① 出征**敌人/掉落预览**(enemy_def + 掉落表已有);
  ② **战力派生**(由 8 维属性计算一个汇总值,纯派生、非新数据)。二者轻量、可增量。
- **Art Spec(主接收方)**:本文 = **全局 UI·juice 轮的布局蓝图**——城镇建筑美术、五板块分区视觉、
  图标/选中态/空位/锁定态/占位文案,均交那一轮统一出。**A 层布局确定后即可上皮**。
- **数据**:无迁移。roster/bag/materials/stages/8 维属性均现有;背包筛选/详情直接遍历现有结构。
- **与 UX-CHANGE-03 收口**:**本文 §3.1 把成员轴的 full 形态由「顶部 tab-row」改定为「左列成员栏」**
  (更宽裕、合参考)。UX-CHANGE-03 仍有效(interim 标签不变、full 仍 deferred 招募),仅**几何细化**;
  已在 UX-CHANGE-03 标注交叉引用。

## 6. 风险与被否选项 / Risks & rejected
- **风险①·城镇窗口几何 → 【已裁决·2026-06-20·SMM:Option A,不改 M3】**:原倾向"城镇用放大工作窗"。
  SMM 核查后裁定**城镇复用 M3.EXPANDED 贴底全宽窗(800×250),不新增几何态**——理由:① 现状 M2.TOWN
  已渲染于 EXPANDED(`enter_game_geometry()`→EXPANDED,TownView/CombatView 同挂 MainArea);② 用户参考图
  为宽幅 ~3.2:1,**正是贴底窗画幅**,本就按此窗设计;③ 为"放大窗"加 M3 第 4 态 + 改 MainArea 舞台结构
  = 违硬 NO「不为没影系统提前抽象」。**条件触发未来债**:若 UI·juice 轮实证 250px 装不下最满板且要城镇
  专属更大窗 → 才 (a) 转 /arch-guard 定 MainArea 尺寸随 flow 变,(b) 回 SMM 给 M3 加第 4 态 TOWN。
  详见 STATE-MACHINES.md §6 #8。**本期 A 层布局一律按 800×250 贴底窗排版。**
- **被否①·照搬参考全部信息(货币/收益秒/每日任务/活动/成就/制作/技能/成功率)**:违硬 NO「不为没影
  系统提前抽象」+ 支柱 1「低负担」。**只取布局骨架,B 层系统项一律标注预留、不建。**
- **被否②·给强化加"成功率/失败"以贴参考**:test-2 强化是**确定性 +1**(守 i7),引入 RNG 失败 = 改玩法
  (Game Designer 的事)且增焦虑(逆支柱 1)。**不做**;参考的成功率区留空或改显"确定 +1"。
- **被否③·把背包并进小队、不做独立板**:材料/分解/任务物品塞进换装条会过载;**独立背包板 + 小队快速
  换装条**分工更清晰(参考亦如此)。
- **风险·一次重排五板块工作量**:A 层是一整轮 UI 工作 → **正好并入 BACKLOG Next① 全局 UI·juice 轮**
  (本就计划"功能做完后统一上皮一轮"),不另起;Producer 若要可按板块切票。
