## 1. 交互一句话 / Overview

常驻 PC 屏幕底部的**悬浮窗挂机打宝游戏**:窗口安静贴底自动战斗,玩家余光扫一眼即懂,
偶尔点开**城镇**主动换装/强化变强,再回**探索**冲更硬的怪。交互重心随支柱 1("伙伴不是任务")
偏向**被动观看 + 低频主动介入**,而非连续操作。

- **主输入设备:键盘 + 鼠标(桌面 Windows)。** 一切导航以鼠标点击为主,热键(F1/F2)为辅。
  无手柄、无触屏路径。
- **窗口几何本身是一种交互态**(异于普通全屏游戏):窗口在「居中较大主菜单窗」、「全宽贴底大窗」
  与「右下角 64×64 小图标」之间切换,是玩家管理"陪伴感强度"与进出系统枢纽的一级动作。

> ✅ **UX-CHANGE-01(09-title-main-menu)已落地(2026-06-20)**:启动落居中主菜单窗、`[☰]` 全局枢纽、
> 设置/退出/覆盖确认四屏、统一 Esc 退一级——均已实现并经人手动 Play 验收 + 代码评审通过。
> 本文原 🔜 标记已转现状。仍带 🔜 的为**后续推广项**(如 Esc 推广到面板/城镇),非本期范围。
>
> 🔜 **UX-CHANGE-02(10-ingame-flow-nav)draft(2026-06-20,待落地)**:把游戏内板块导航收成
> **城镇(家·枢纽)↔ 探索(派出挂机)**,**新游戏/继续都落城镇(暂停)**(用户拍板·有意接受支柱 1 让位)。
> 城镇升为**枢纽 + 四覆盖式子板块**(工匠 / 小队 / 酒馆〔占位〕/ 出征〔关卡选择〕);背包拆解
> (探索=掉落预览 / 城镇=小队换装 + 工匠);新增**「待回城」标记**(战斗中点回城不打断,本关结算后返城);
> EXPLORE↔TOWN 收进 GameFlow + Esc 推广到游戏内板块。本文下列带 🔜·UX-02 的是**目标态、尚未实现**;
> 现状仍是"探索为隐式家、城镇单层、进出城由 TownView 自管"。详见 `harness/ux/UX-CHANGE-02-ingame-flow-nav.md`。

## 2. 屏幕与状态地图 / Screen & flow map(心脏)

**两套正交的状态机**(现状如此,但从未被显式建模——见 §6 债 #7):

### A. 窗口几何态(整个 shell 级,由 `floating_shell.gd` 驱动;MENU 由 GameFlow 命令)
```
[启动] ──> MENU(居中较大主菜单窗,560×400)──继续/新游戏──> 收缩 ──> EXPANDED(全宽贴底 800×250)
EXPANDED ──F1 / 点"收起"按钮──> COLLAPSED(右下角 64×64 handle 图标)
COLLAPSED ──F1 / 点 handle──> EXPANDED
EXPANDED / Town ──点右上[☰]──> 放大 ──> MENU(记来源态) ──继续/Esc──> 回来源态(EXPLORE/TOWN)
MENU(退出)──> 〔退出确认〕──确定──> autosave ──> quit
EXPANDED/COLLAPSED ──F2──> (切换窗口置顶 always-on-top,无屏变化,仅窗口标志位)
```
- 收起态:仅显 handle 图标;主区内容全隐,**后台模拟照跑**(固定步长 tick 不停)。
- MENU 态:放大的系统枢纽,**不暂停 sim**(守支柱 1 后台推进,与收起同理;`open_menu`/`_resume_to_source`
  绝不动 `arena.running`)。MENU 态本身无收起出口(F1 无反应)。
- **部分正交(已落地)**:`{EXPANDED,COLLAPSED}` 与游戏态 `{EXPLORE,TOWN}` 正交(游戏中可随意收起);
  但 **`MENU` 几何 ↔ 主菜单内容 1:1 锁定**——收起(F1/handle)仅 EXPLORE/TOWN 可用,主菜单态无收起出口
  (离开主菜单只能 继续/新游戏/退出)。由 GameFlow 命令 M3 切换(STATE-CHANGE-01 §3.3,债 #7 闭合)。

### B. 主区/窗口内容态(主菜单四屏挂在 `GameFlow` 节点下;探索/城镇挂 `MainArea` 下)
```
[启动] ──> 主菜单 Title ──[继续/新游戏]──> 探索/战斗 Explore   〔继续=续战游标(无档置灰);新游戏=有档先确认覆盖〕
Title ──[设置]──> 设置 Settings ──返回/Esc──> Title
Title ──[退出]──> 〔退出确认〕──取消/Esc──> Title / ──确定──> autosave→quit
Explore ──点"进城"──> 城镇 Town ──点"出城"──> Explore
Explore ──点"背包/装备"──> 背包/装备只读面板(Explore 内模态遮罩) ──再点──> Explore
Explore / Town ──点右上[☰]──> 主菜单(MENU_OVERLAY)──[继续]/Esc──> 回来源态(Explore 续战 / Town 仍暂停)
MENU_OVERLAY ──[新游戏]──> 〔覆盖确认〕──确定──> 新局 Explore     〔游戏中也能开新档〕
团灭 ──(无屏切换,仅日志"团灭·回退刷怪" + 自动回退刷怪)──> Explore   〔有意:无死亡屏,见 §6 #5〕
关程序(OS 关窗) ──autosave──> 退出
```
- `进城` = `Game.pause_run()` 冻结挂机 + 隐 CombatView + 显 TownView 内容;`出城` 反之 `resume_run()`(不回血)。
- `背包/装备面板` = CombatView 内的全区遮罩(`MOUSE_FILTER_STOP` 吃点击),纯只读查阅,不暂停战斗。
- `主菜单四屏`(主/设置/覆盖确认/退出确认)= `GameFlow` 自管 `.visible`,覆盖在底层视图之上;MENU 几何下
  shell 已隐 `main_area`,菜单屏 bg 又近不透明,故底层视图态被冻结、resume 时原样恢复(GameFlow 不跟踪城镇态)。
- **进城与背包面板互不感知**:城镇态下战斗暂停,背包面板仍是战斗态的产物——两者仍无统一"返回"约定(🔜 Esc 推广待办,债 #3)。
- **暂停规则(UX-CHANGE-01 已落地)**:仅 Town(改装备)暂停挂机;Menu(MENU_OVERLAY)/Collapse **不暂停**。

> 🔜 **UX-02 目标态(待落地,delta vs 上图)**:顶层游戏内板块收成 **城镇(家·枢纽)↔ 探索(派出挂机)**;
> **BOOT/继续/新游戏 落点由"探索(running)"改为"城镇枢纽(暂停)"**,经出征→「出击」才开打(回城=pause、出击=resume)。
> 城镇升为**枢纽 + 四覆盖式子板块**:工匠(分解/强化/制作)/ 小队(队员·换装)/ 酒馆(占位)/ 出征(关卡选择→出击)。
> EXPLORE↔TOWN **上移进 GameFlow**(收债 #4 残留)。背包拆解:探索内「背包/装备」→**掉落预览**(只读·掉落装备+材料·不暂停);
> 城镇换装→小队子板块、强化→工匠子板块(整合材料)。**新增「待回城」标记**:战斗中点「回城」不打断,按钮转
> 「已请求回城,本关结束后返回」,本关结算后再 pause 返城(同向不变量 #12 波清空粒度 + 支柱 1 不打断)。
> Esc 推广到游戏内板块(子板块→城镇枢纽 / 掉落预览→探索 / 探索→城镇 / 城镇枢纽→主菜单)。
> **支柱 1 权衡**:落静止暂停城镇削弱"启动即陪伴",用户有意接受。**状态三层多数已由四层地基承载,不重造**(真 delta = 掉落暂存 + 待回城标记)。详见 `UX-CHANGE-02`。

## 3. 每屏职责与交互状态 / Per-screen purpose & states

### 探索/战斗 Explore(`combat_view.gd`)
- **职责**:展示自动战斗当前态——小队血条、敌人(单/多敌波)、进度读出、战斗日志、掉落 FX。纯只读渲染,收起时隐藏、模拟照跑。
- **交互态**:
  - `default / grinding`:正常刷怪;无按钮强调。
  - `推进可用(GRINDING)`:右上「推进」按钮显形 + 卡关时下方「回城变强」邀请行(克制、不弹窗、可无视)。
  - `通关倒计时(STAGE_CLEAR_COUNTDOWN)`:中央倒计时读出"通关!Xs 后推进"。
  - `敌人狂暴(enraged)`:右上狂暴横幅常驻。
  - `里程碑(末关通关)`:更亮更久的全屏占位闪。
  - **error**:"(无 Game 单例)"——装配失败兜底文案。
- **可点元素**:「推进」「进城」(由 TownView 提供,叠在本屏)「背包/装备」「收起」。

### 背包/装备只读面板(Explore 内,`combat_view.gd::_build_panel`)
- **职责**:左栏掉落包 + 右栏当前装备 & 8 维属性明细;掉落自动填空时右栏绿闪("变强看得见")。只读,不可操作装备。
- **交互态**:`有内容` / `掉落包为空"(空)"` / `未进入战斗"(未进入战斗)"`。
- **关闭**:再点「背包/装备」按钮(无 Esc)。

### 城镇 Town —— 枢纽 + 五子板块(`town_view.gd`;布局蓝图见 🔜 UX-CHANGE-04)
- **职责**:支柱 2 主动构筑落点——暂停挂机后管理小队/装备/强化/出征。**v1 现状 = 最小占位线性控件**;
  目标布局(按用户参考图)= 各板块统一三区/左中右 IA,记于 🔜 **UX-CHANGE-04**,并入全局 UI·juice 轮。
- **窗口几何(已裁决·2026-06-20·SMM)**:城镇**复用 M3.EXPANDED 贴底全宽窗**(800×250),**不新增几何态**——
  现状 M2.TOWN 已渲染于 EXPANDED(`enter_game_geometry()`→EXPANDED;`main_area.visible` 仅 EXPANDED 真,
  TownView/CombatView 同挂 MainArea),且参考图为宽幅 ~3.2:1,正是贴底窗画幅。**条件触发未来债**:若 UI·juice 轮
  实证 250px 装不下最满板且要城镇专属更大窗 → 才 (a) 转 /arch-guard 定 MainArea 舞台尺寸随 flow 变,(b) 回 SMM
  给 M3 加第 4 态 TOWN。本期不做。详见 STATE-MACHINES.md §6 #8。

**🔜 子板块目标布局(UX-CHANGE-04;✅=映射现有系统 / ⚠=仅参考·略或缓)**:
- **枢纽 Hub**:中央 4 座建筑 = 入口(打造/酒馆/小队/出征)✅;顶右 `[☰]`+⚙ ✅;底左 头像+名+Lv ✅(+战力派生 flag GD);

### 收起态 Collapsed(`floating_shell.gd`)
- **职责**:把陪伴感降到最低——仅右下角一个 handle 图标,降帧到 15fps,模拟照跑。
- **交互态**:`default`(仅 handle 可点/可按 F1 恢复)。

### 主菜单 Title / Main-Menu(MENU 居中窗,UX-CHANGE-01 / STATE-CHANGE-01;已落地)
- **职责**:游戏前门 + 唯一系统枢纽。同一屏出现在**两个流程语境**(FSM 态)——
  `TITLE`(启动落点,无来源)与 `MENU_OVERLAY`(游戏中经 `[☰]` 调回,带 `menu_return_to` 来源寄存器)。
  玩家可见差异仅:「继续」去哪(TITLE→续战/新局;OVERLAY→回来源态)+ 两者都**不暂停 sim**。
  **设置/新游戏/退出在两语境均可达**(游戏中也能开新档,经覆盖确认)。
- **交互态**:`default·有存档`(继续=主操作 / 新游戏 / 设置 / 退出)/ `default·无存档`(继续 disabled 或隐藏,新游戏=主)/ `〔覆盖存档确认〕`(新游戏撞上已有档)/ `〔退出确认〕`。
- **可点元素**:继续、新游戏、设置、退出。
- **返回 / Esc**:子层(设置/确认)`Esc` 退一级;**顶层 `MENU_OVERLAY` 的 `Esc` = 继续**(回来源,符合"退一级");顶层 `TITLE` 为根屏、**无 Esc 出口**。

### 设置 Settings(Title 子屏,UX-CHANGE-01;已落地为占位屏)
- **职责**:系统选项落点(补债 #2/#6)。当前为**占位屏**(单行文字 + 返回按钮);音量/键位重映射/关于待 Art Spec+后续填实。
- **交互态**:`default`。**返回**:`Esc` / 返回按钮 → 主菜单。

## 4. 输入·导航·反馈约定与不变量 / Conventions & invariants

**现状(多为隐式、未统一,记录现实)**:
- **焦点模型**:纯鼠标点击,无键盘焦点环、无 Tab 导航、无手柄。
- **热键**:`F1` = 收起/展开;`F2` = 置顶切换。**仅窗口聚焦时生效**,硬编码不可改(债 #6)。
- **back/cancel**:**`Esc` 统一返回上一层已落地于主菜单域**(UX-CHANGE-01):设置/覆盖确认/退出确认层
  `Esc` 退一级回主菜单;顶层 MENU_OVERLAY 的 `Esc` = 继续(回来源);顶层 TITLE 为根屏无 Esc 出口。
  **面板/城镇尚未纳入**(仍靠"再点同一按钮"/"出城"关)——后续推广待办(债 #3 部分解)。
- **反馈**:伤害飘字 / MISS 飘字 / 战斗日志(滚动 4 行)/ 蓝金掉落光柱 + 金装窗内一闪 / 换装对比绿↑红↓ / 自动填空绿闪。
  反馈一律**限窗口内**——不弹 OS 通知、不抢焦点、不打断(守支柱 1)。

**应当始终成立的不变量(部分现已破,标注)**:
- ✅ **反馈不抢焦点 / 不弹 OS 通知 / 不强迫操作**(支柱 1 红线)——现状满足。
- ✅ **收起态模拟照跑**——现状满足。
- ⚠ **"任何屏都有出路、无死胡同"**——现状基本满足(每个面板/城镇都有关闭按钮),但**仍依赖鼠标**;
  无键盘兜底。主菜单/设置/确认层已确立 `Esc`(已落地);面板/城镇尚未纳入。
- ✅ **"窗口几何态 × 主区内容态"——由"全正交"改为"部分正交"已落地**(UX-CHANGE-01 / STATE-CHANGE-01):
  `{EXPANDED,COLLAPSED}` 与 `{EXPLORE,TOWN}` 正交;`MENU` 几何 ↔ 主菜单内容 1:1 锁定(主菜单态不可收起,
  F1 无反应)。由 GameFlow 命令 M3 切换,人手动 Play 验收"反复 MENU↔贴底切换无抖动"通过(债 #7 闭合)。

## 5. 信息架构 / HUD / 可达性

- **常驻 HUD(探索态)**:左上小队血条行(名字+血条+数值,4 格结构 v1 填 1)、左上进度读出("第 X 关·场景 Y/3 / Boss")、左下滚动战斗日志、右上「推进」/状态横幅。
- **菜单信息层级**:城镇为三栏(左槽位 / 中对比·强化 / 右背包);背包面板为双栏(左掉落包 / 右装备+属性)。均为占位程序美术,层级靠位置而非视觉权重。
- **可达性:近乎零基建**(债 #6)——
  - 键位不可重映射;无字体大小选项;窗口尺寸固定(仅全宽/图标两态)。
  - **多处仅靠颜色编码信息无冗余**:稀有度白/蓝/金、近战红/远程蓝占位块、对比绿↑红↓——色盲不可辨。
  - 无音频(占位待补),故无"声音替代视觉"路径。
