# 游戏架构设计文档：2D 横版挂机 ARPG

## 1. 核心架构理念

本项目的核心目标是构建一个能够支撑复杂暗黑类数值与 PoE 级海量词缀装备的挂机游戏架构。为避免后期代码臃肿与逻辑耦合，确立以下三大基本原则：

* **数据驱动 (Data-Driven)**：将策划内容（种族、装备基础、海量词缀）彻底剥离出代码，使用标准 JSON 格式进行维护与继承。
* **组合优于继承 (Component-Based)**：废弃面向对象中实体类的深层继承（如 `Actor -> Monster -> Boss`）。所有实体皆为“空壳”，能力由挂载的独立组件决定。
* **表现与逻辑分离**：底层 C# 数值计算、状态机流转与上层的 2D 动画播放、特效展示严格解耦，确保游戏在无头（Headless）挂机模式下依然能精准进行数值演算。

---

## 2. 核心数据流向设计

为实现极高的复用率，游戏数据被严格切分为“静态模板”与“动态实例”两层。

| 数据层级 | 存储形态 | 核心职责 | 特点 |
| :--- | :--- | :--- | :--- |
| **基础模板层** | 静态 JSON 文件 | 定义世界上存在什么。例如：铁剑的基础属性范围、所有可能出现的词缀库及其出现权重。 | 只读不写，游戏启动时加载到内存，作为生成器的蓝图。 |
| **运行时实例层** | C# 内存对象 | 定义玩家当前拥有的具体物品。例如：玩家打怪掉落的那把带有特定词缀的“极品铁剑”。 | 动态生成，包含独立状态，通过存档系统进行序列化保存。 |

---

## 3. 高弹性属性计算系统 (Stats System)

角色的最终战力由自身基础属性与装备词缀共同构成，需设计支持修饰器（Modifiers）的动态计算系统，避免硬编码加减法。

### 核心计算公式

对于生命值、攻击力、攻击速度等任何单项属性，采用统一的分层级计算公式：

$FinalStat = (BaseValue + \sum FlatModifiers) \times (1 + \sum PercentModifiers)$

### 属性系统工作流

* **注入加成**：玩家穿上装备时，装备组件将词缀附带的 `FlatModifier`（固定值）和 `PercentModifier`（百分比）注入到角色的 `StatsComponent` 中。
* **动态结算**：任何属性查询操作（如攻击时获取当前攻击力）都会基于上述公式实时（或脏标记缓存）计算出最终结果。
* **无损卸载**：玩家脱下装备时，直接从修饰器列表中移除对应的 Modifier，属性自动回退，绝不产生数值残留 bug。

## 4. PoE 级装备生成流水线 (Item Pipeline)

摒弃手动配置每一件成装的低效做法，建立基于权重的动态生成流水线。

### 装备生成步骤

1.  **确定基底 (Base Item)**：根据怪物等级与掉落列表，生成一件基础物品（如：`ilvl = 30` 的普通长剑）。
2.  **随机稀有度 (Roll Rarity)**：根据全局掉落率设定，通过随机数决定物品品质（白、蓝、金）。
3.  **抽取词缀 (Roll Affixes)**：从 JSON 词缀库中过滤出 `min_ilvl <= 30` 的合法词缀。根据词缀的 `weight`（权重）进行随机池抽取，填满当前品质允许的前后缀数量限制。
4.  **生成实例 (Instantiate)**：将抽取的词缀数值写入基础物品对象，生成唯一的运行时装备实例。

## 5. 组件化实体拆分 (Entity Components)

战场上的所有单位（玩家、怪物）共享同一套底层的空壳结构，通过挂载以下核心组件实现业务逻辑。

* **StatsComponent (数值组件)**：管理所有基础属性与修饰器，对外提供最终数值的读取接口。
* **EquipmentComponent (装备组件)**：管理各装备插槽（头部、躯干、武器等），负责监听装备的穿脱事件，并向 `StatsComponent` 派发或回收修饰器。
* **SkillComponent (技能组件)**：读取最终战斗数值，负责普攻与特殊技能的范围判定、冷却计算与伤害结算。
* **AutoAIComponent (挂机 AI 组件)**：轻量级状态机，负责驱动寻敌、移动、攻击、拾取等挂机行为循环。
* **AnimationComponent (动画组件)**：监听 AI 和技能组件的事件（如：触发攻击动作），纯粹负责 2D 美术资产的表现，不参与任何数值计算。

## 6. 项目物理目录规划

强制规范项目目录，严禁跨模块反向依赖。

| 路径 | 内容类型 | 职责说明 |
| :--- | :--- | :--- |
| `GameData/Config/ItemBases.json` | 静态数据 | 基础装备模板（如白板长剑的内置属性）。 |
| `GameData/Config/AffixPool.json` | 静态数据 | 词缀库（ID、类型、权重、最低等级、加成数值）。 |
| `GameData/Config/MonsterBase.json` | 静态数据 | 怪物基础模型数据，使用 `inherit_id` 实现差异化继承。 |
| `Scripts/Core/Components/` | C# 逻辑代码 | 存放上述定义的所有核心组件（Stats, Equipment 等）。 |
| `Scripts/Core/Systems/` | C# 逻辑代码 | 存放全局系统，如 `LootGenerator`（装备生成器）与 `CombatResolver`（战斗结算器）。 |
| `Assets/Art/Entities/` | 美术资产 | 存放单位分类的 PNG 序列帧，统一命名规范以支持动态加载。 |

> **开发备忘录**：
> 在开始编写 UI 和引入美术资源之前，务必优先完成 `LootGenerator` 和 `StatsComponent` 的纯代码跑通。建议使用控制台输出的形式，模拟打怪 -> 掉落金装 -> 穿戴 -> 攻击力提升的完整数据流，以此验证核心架构的健壮性。
