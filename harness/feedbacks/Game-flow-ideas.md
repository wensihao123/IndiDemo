# 挂机打怪掉宝小游戏 — UX Flow & 状态管理设计

> 一个「Idle/放置 + 装备驱动」的挂机打怪游戏。核心是把 **导航(界面树)**、**持久存档(进度/仓库/队伍)** 和 **临时战斗状态** 这三层分清楚。

---

## 一、界面导航树

```
主菜单
  └─(开始 / 读档)→ 城镇  ← 待机主界面 / 枢纽
                    ├─ 工匠   打造系统(分解 / 强化 / 制作)
                    ├─ 酒馆   招募系统(第一版占位)
                    ├─ 小队   队员 · 装备管理
                    └─ 出征   关卡选择
                              └─ 关卡战斗(推关 · 掉落)
                                   └─(关卡完成后可回城)→ 城镇
```

整体流程:**开始 → 城镇(整备、打造、编队、调整装备)→ 出征 → 回城**,循环往复。

---

## 二、UX Flow

### 主菜单 → 城镇
进入时区分「新开档」和「读档」:
- **新开档**:初始化默认存档(空仓库、初始队员、关卡 1)。
- **读档**:恢复上次保存的状态。

### 城镇(待机主界面)
作为枢纽,四个板块都遵循「进入子界面 → 操作 → 返回城镇」的模式。任何子界面都不改变玩家在世界中的「位置」(玩家始终停留在城镇),只是叠加的面板。

> 建议:子界面用 **覆盖式面板** 而非完全跳转,返回成本低、状态保留自然。

### 工匠(打造)
- **布局**:左仓库 / 右打造。
- **交互核心**:选中 → 操作。从左侧选一件装备或材料,右侧根据当前标签(分解 / 强化 / 制作)显示可执行操作和消耗预览,确认后即时更新仓库。
- **关键体验点**:操作前展示 **消耗与结果预览**(消耗哪些材料、强化成功率、产出什么),避免误操作。

### 小队(队员管理)
- **布局**:左仓库(按部位分标签页)/ 右人物详情(装备栏 + 详细属性)。
- **交互核心**:选队员 → 看装备栏和属性 → 从左侧拖 / 点装备到对应部位。
- **关键体验点**:穿脱装备即时重算并 **高亮属性变化**(如 攻击 +12)。与工匠共用同一份仓库数据,只是过滤展示方式不同。

### 酒馆(招募)
第一版只放 **占位界面**(标题 + "敬请期待"),但导航入口和返回逻辑先接通,方便后续补内容时不动框架。

### 出征(打怪)
选关卡 → 进入战斗。战斗是 **自动循环(挂机)**,界面显示当前关卡、队伍血量、战斗日志、掉落暂存。

两个回城路径要分清:

| 路径 | 行为 |
|------|------|
| **关卡完成后回城** | 正常出口,直接结算回城。 |
| **战斗中点「回城」** | 不立即中断,设一个 `待回城` 标记,等当前关卡打完再结算返回。按钮变为「已请求回城,本关结束后返回」状态。 |

## 三、状态管理需求

把状态分成三层管理,这是这类游戏最关键的设计决定。

### 1. 持久层(存档,需要落盘)

游戏的真实进度,任何修改都应能被保存:

- **玩家档案**:货币、当前最高关卡、已解锁板块。
- **仓库(单一数据源)**:所有装备和材料的唯一列表。装备建议存为对象数组,每件带:
  ```
  { id, 部位, 品质, 基础属性, 强化等级, equippedBy }
  ```
  其中 `equippedBy` 表示装备在谁身上,未装备则为空。
  > **不要把装备复制进队员的装备栏。** 队员只存「装备 id 引用」,仓库是唯一真相。这样穿脱、分解、强化都只改一处,避免数据不一致。
- **小队**:队员列表。每个队员存基础属性 + 各部位装备的 **id 引用**,**不存** 计算后的总属性。

### 2. 派生层(计算得出,不存储)

- **队员最终属性** = 基础属性 + 所有已装备的加成 → 应 **实时计算或带缓存的派生值**,不存进存档。否则强化一件装备后,得手动同步所有引用它的地方,极易出 bug。
- 仓库的标签页过滤、可制作列表等也都属于派生层。

### 3. 临时层(战斗运行时,不一定持久化)

- 当前是否在战斗、当前关卡、推进进度、队伍当前血量、战斗日志。
- **掉落暂存区**:掉落先进暂存区,在「关卡结算」这一刻才合并进持久仓库。
- `回城请求标记`。

> 中途异常退出不会污染存档,也给"本关打完才回城"提供了天然的结算点。

---

## 四、反复踩坑的点

- **存档时机**:Idle 游戏适合「关键节点存(回城、打造、关卡结算)+ 定时自动存」双保险。
- **结算是唯一写入口**:战斗过程别直接写仓库,统一在关卡结算时把暂存掉落、经验、进度一次性提交,逻辑清晰且可回滚。
- **城镇态 vs 战斗态隔离**:进城镇时战斗临时状态应被清空或归档,避免残留状态串味。
- **数据驱动**:板块解锁、关卡数据建议做成配置表,酒馆这类占位系统后续才好平滑接入。

---

## 五、状态分层速查

| 层 | 内容 | 是否落盘 | 写入时机 |
|----|------|:--------:|----------|
| 持久层 | 玩家档案、仓库、小队(引用) | 是 | 关键节点 + 定时 |
| 派生层 | 队员最终属性、过滤/可制作列表 | 否 | 实时计算 |
| 临时层 | 战斗进度、当前血量、掉落暂存、回城标记 | 否(结算后提交) | 关卡结算 |
