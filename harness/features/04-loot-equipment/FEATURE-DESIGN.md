# FEATURE-DESIGN — 暗黑式掉落与装备 (Loot & Equipment)

> 承 IDEA(2026-06-18 重刷版)。本文把 IDEA §8 全部线头收敛为**决策**或**flag**。
> 用户已于 2026-06-18 对关键决策点逐条拍板(见 §0 决策来源),其中 **B-4 用"物品等级 + 词缀分阶池"
> 取代了 IDEA §4 的"词缀通吃 6 维"模型**——本文以 B-4 为准。
> 仅设计意图 + 占位数值;不写实现/资源/排期。所有数值标 `占位`,精调进 §8 F1 数值专章。

## 0. 决策来源(2026-06-18 用户拍板)
- **A-1** 战士开局 = **自带全白基础装,唯独饰品(项链)槽空**。
- **A-2** = (a):**04 只做只读掉落包;支柱 2"变强爽感"承认要等 05 才完整**,04 阶段只验"掉落/分解/自动填空/只读查阅"机制跑通。**04 不做任何临时换装入口。**
- **A-3** 分解门槛默认 = **白色全分解**;自定义分解条件留待以后(05/更后)扩展。
- **B-4** 词缀模型 = **物品等级(ilvl)+ 按部位划分的词缀池 + 词缀分阶(Tier)**(PoE 式)。**取代 IDEA §4 的"通吃 6 维"。**
- **B-5** **稀有度只与词缀条数挂钩**(白 0 / 蓝 1-2 / 金 3+);稀有度**不**抬数值,数值高低只由 Tier(受 ilvl 限制)决定。
- **C-6** 材料 = **按部位分 3 种(武器 / 护甲 / 饰品)× 每种白/蓝/金 等级**。
- **C-7** 背包**暂不设上限**,以后完善。

## 1. 玩家体验 / Fantasy
"我挂着它刷。开局战士裸着脖子——第一件项链'叮'地自动戴上了,肉眼可见结实了点。之后白装哗哗分解成材料、攒着;
偶尔点开掉落包翻翻:'掉了把金武器、三条词缀带双暴击,物品等级还挺高!'——记一笔,等回城(05)再细挑换上。"
洪水自消化(白装自动分解)、零必做操作(挂机也在攒材料 + 填了空槽)、参与感留给"回城时少数好货的取舍"(05)。
04 自身 = **看着小队自动武装起一点 + 翻看战利品的小确幸**;主动构筑的高光归 05。

## 2. 核心循环 / Core loop
1. **挂机产出(自动,承 02):** 击杀 → `Combat.loot_dropped(kind, rarity)` → 04 据 kind=`equipment`/`material`/`gold` 与 rarity 生成实例。装备实例 = 部位 + ilvl + 基底招牌轴 + N 条词缀(N 由稀有度定)。
2. **进包前过滤(自动):** 装备实例先问"有没有对应空槽"——
   - **有空槽 → 自动穿上(填空优先,见 §4.4,白装也照穿)。**
   - **无空槽 → 按门槛过滤:** 白(默认门槛)→ **自动分解成对应部位材料**;蓝/金 → **进背包(掉落包)**。
3. **攒材料(自动):** 自动分解产出"部位材料(白/蓝/金)",累加进材料库存,发 `material_gained` 事件(喂 05 经济)。
4. **只读查阅(手动,04 唯一玩家介入 = 仅"看"):** 悬浮窗点开**掉落包面板** → 浏览包内每件的部位/ilvl/基底/词缀(分稀有度色)。**只能看,不能换。**
5. **变强 → 回循环:** 04 内变强 = 那次空槽自动填充;持续变强(手动换上掉落包里的好货)= **05 回城换装**。更深的怪 → 更高 ilvl → 能掉更强 Tier 的词缀。

> **边界(B1,Producer 已追认):** 04 = 掉落 + 物品数据(基底+词缀+ilvl)+ 自动分解 + 自动填空 + 材料产出 + 只读掉落包。
> **手动换装 + 对比面板(绿↑红↓)+ 打造/强化消耗材料 + 自定义分解条件 = 05 城镇。**

## 3. 关键规则与状态 / Key rules & states

### 3.1 三槽位 + 招牌基底(承 03 §3.7,不变)
| 槽位 | 招牌基底轴(给身份) |
|------|----------------------|
| 武器 weapon | 攻击 attack + 攻速 attack_speed |
| 护甲 armor | 护甲 armor(减伤) |
| 饰品 accessory(项链) | **生命 / 闪避 / 秒回 三选一**(每件掉落时随机定一个招牌轴) |

每槽 1 件。基底招牌轴的**数值随 ilvl 提升**(把基底当作该部位一条"隐式 T 阶词缀",用同一套 Tier 机制表达;曲线留 §8 F1)。

### 3.2 装备实例数据模型(B-4)
一件装备 = `{ 部位, 物品等级 ilvl, 基底(招牌轴 + 该轴数值), 词缀[] , 稀有度 }`。
- **ilvl(物品等级):** 掉落时由掉落来源给定(见 §8 **F-A**:ilvl 来源待接线)。ilvl **限制能 roll 到的最低 T 阶编号**(越高 ilvl 解锁越强的 Tier)。
- **稀有度(B-5):** 仅 = 词缀条数。**白 = 0 词缀(仅基底)/ 蓝 = 1-2 词缀 / 金 = 3+ 词缀。** 稀有度不影响任何数值大小。
- **词缀:** 每条 = `{ 词缀类型, 所在 T 阶, 该 T 阶区间内 roll 出的具体值 }`。

### 3.3 按部位的词缀池(B-4,取代 IDEA"通吃")
词缀类型对应 03 的战斗维度:攻击 attack / 攻速 attack_speed / 护甲 armor / 闪避 dodge_chance / 暴击率 crit_chance / 暴击伤害 crit_mult / 秒回 hp_regen / 生命 max_hp。
**暴击率与暴击伤害是两条独立词缀**(收敛 IDEA §8#1 的"率 vs 倍率"= 两者皆可,各算一条)。

| 部位 | 词缀池(可 roll 的词缀类型) | 主题 |
|------|------------------------------|------|
| 武器 weapon | 攻击 / 攻速 / 暴击率 / 暴击伤害 | 攻击系 |
| 护甲 armor | 护甲 / 闪避 / 生命 / 秒回 | 防御系 |
| 饰品 accessory | 生命 / 护甲 / 秒回 / 暴击率 / 暴击伤害 / 攻击 / 攻速 | 混合(用户给定示例) |

> 注:饰品池按用户示例列(未含"闪避")。若数值专章为对称想补"闪避"入饰品池,属低风险微调,留 §8 F1。
> 经典暗黑"武器上偶尔出血量词缀"的跨主题惊喜,本版**收敛为按部位池**——武器只出攻击系、护甲只出防御系,饰品才是混合大杂烩。是有意取舍(可读 > 全随机)。

### 3.4 词缀分阶(Tier)+ ilvl 门槛(B-4 核心机制)
- 每条词缀类型有自己的一串 T 阶:`T_n`(最弱,编号最大)→ … → `T1`(最强,编号最小)。**不同词缀的阶数可不同**(用户例:生命 T10→T1 共 10 阶;暴击率 T5→T1 共 5 阶)。
- 每个 T 阶 = **一个数值区间** + **一个 ilvl 解锁门槛**(`ilvl ≥ 该阶门槛`才可能 roll 到)。
- **roll 一条词缀:** 在该部位词缀池里选一个词缀类型 → 取该词缀**所有 ilvl 门槛 ≤ 物品 ilvl 的合格 T 阶** → 在合格阶里挑一阶 → 在该阶区间内 roll 具体值。
- **同一件上同一词缀类型不重复**(一件最多每种词缀一条)。
- 合格阶的挑选权重(均匀?偏向新解锁的强阶?)留 §8 F1;v1 占位 = **合格阶内均匀**。

**占位 Tier 表(数值全是占位,结构才是交付物;精调进 §8 F1):**

| 阶 | 值区间(占位) | ilvl 门槛(占位) |
|----|----------------|-------------------|
| T10 | +1~5 | 1 |
| T9 | +5~10 | 8 |
| T8 | +10~18 | 16 |
| T7 | +18~28 | 25 |
| T6 | +28~42 | 34 |

暴击率 crit_chance(%,5 阶):

| T5 | 0.5%~2% | 1 |

> 其余词缀(攻击/攻速/护甲/闪避/暴击伤害/秒回)按同样结构各自建表(阶数、区间、门槛由 §8 F1 数值专章填),此处不逐一占位以免误当定稿。

### 3.5 开局装备(A-1)
战士开局自带:**武器(全白基底,ilvl=1)+ 护甲(全白基底,ilvl=1),饰品槽空。**
→ 结果(有意):04 阶段唯一会触发的"自动填空"是**第一件饰品掉落填进空项链槽**;之后三槽满,后续掉落走分解/进包。
**这是 A-1 + A-2(a) + "绝不替换已穿戴"三条共同的直接后果**:04 的"挂机变强"≈ 那一次项链填充;持续变强靠 05 换装。开局空装的数值占位留 §8 F1。

### 3.6 换装 / 穿戴规则
- **04 内只有"自动填空槽":** 空槽 + 对应部位掉落 → 自动穿上;**绝不自动替换已穿戴的。**(理由:词缀/未来套装让"哪件更好"无法机械判定,纸面比大小可能变强实则变弱。)
- **自动填空只在掉落瞬间触发**:用"刚掉的这件"填空,不扫包做优选(优选 = 05 的事,且需强度评估,本版刻意不做)。
- 手动换装 + 逐项差值对比 = **05**。

### 3.7 自动分解 → 材料(A-3 / C-6)
- **门槛(A-3):** 默认"稀有度 ≤ 白 → 自动分解"。门槛值走 Resource/`@export`(**不硬编码**,守 hard-NO),但自定义分解条件 UI 留 05/更后。
- **产出(C-6):** 一件分解 → 产出**该部位材料**(武器材料 / 护甲材料 / 饰品材料)**× 该件稀有度等级**(白件→白材料、蓝→蓝、金→金)。
  → 04 默认只分解白件,故默认只产**白材料 × 3 部位**;蓝/金材料只有 05 调宽门槛后才出现(数据结构先支持白/蓝/金三档,避免日后重写)。
- **材料库存:** 04 维护 `{部位 × 稀有度 → 数量}` 计数 + 发 `material_gained(slot, rarity, amount)` 事件。**04 只产+存材料;消费(打造/强化)= 05。**(类比 02→04 的 `loot_dropped` 边界。)
- 单件产几个材料的公式留 §8 F1(占位 = 每件白装产 1 个对应白材料)。

### 3.8 填空 vs 分解的优先级(本文新定,解 B-4 隐含冲突)
**空槽白装冲突:** 一件白装掉落、其部位槽位为空 → "自动填空(§3.6)"要它穿上、"自动分解(§3.7)"要它变材料,二者矛盾。
**裁定:自动填空优先。** 空槽遇到对应部位掉落(**含白装**)→ 先穿上;**只有该部位已穿戴时,白装才走自动分解。**
理由:有总比没有强(开局空项链该被第一件白项链补上,而非分解掉永远空着);也让 A-1 的开局空项链能尽快"叮"一下,兑现 §1 fantasy。

### 3.9 背包 / 掉落包(C-7)
- 掉落包 = 一个朴素列表(蓝/金进包的装备)。**暂不设上限**(C-7);满包兜底留以后(§8 收敛 IDEA#8:暂不做)。
- 只读:能看,不能换、不能手动分解(手动操作归 05)。

### 3.10 与战斗的接口(承 03,已通)
穿戴的 3 件(基底招牌轴 + 各自词缀)汇总成战士的 8 项战斗数值,写进 `PartyMember`(attack / max_hp / attack_speed / armor / dodge_chance / crit_chance / crit_mult / hp_regen),喂 03 扩好的解算。汇总 = 各来源同维相加(占位规则;封顶/递减留 03 已有机制 + §8 F1)。

## 4. 反馈 / juice 意图 / Feedback & juice intent
- **掉落瞬间(已有,02):** 蓝/金出对应色光柱;白装默默(现在多半直接分解)。
- **自动填空时:** 轻提示(战士该槽位"叮"地亮一下 + 一行日志"自动装备:蓝护甲 ilvl 23")。**不弹 OS 通知**(守支柱 1)。04 内这基本只在第一件饰品时出现一次(§3.5)。
- **自动分解时:** 极轻(材料计数角标 +1 或攒批后一行汇总日志),不逐件打扰。
- **进包提醒:** 有蓝/金进掉落包 → 角标/微光,不打扰。
- **掉落包面板(只读):** 列每件 部位 / ilvl / 基底招牌轴 / 词缀(含 T 阶可选显示)/ 稀有度色。**04 不做"对比已穿戴"的绿↑红↓**——那是 05 的核心反馈。04 只回答"我刷到了什么",不回答"该不该换"。
- **自动填空后:** 战士属性数字跳变(间接:怪死更快/血条更厚)。主反馈 = 属性 + 提示,不强求战斗演出。

## 5. 成功标准 / Success criteria(04 切片可独立验收的点)
1. **掉落生成正确:** `equipment` 掉落能据部位生成 基底招牌轴 + 据 ilvl 限制的合格 T 阶 roll 出 N 条池内词缀,N 符合稀有度(白0/蓝1-2/金3+)。**可 gdUnit4 纯逻辑测**(roll 边界:ilvl 卡门槛、词缀不重复、部位池约束、稀有度→条数)。
2. **填空优先于分解:** 空槽掉对应部位(含白)→ 穿上;已穿戴时白件→分解出材料。**可 gdUnit4 测。**
3. **材料正确累加 + 发事件:** 分解白件 → 对应部位白材料 +N,`material_gained` 触发。**可 gdUnit4 测。**
4. **属性汇总进战士:** 穿戴变化后 `PartyMember` 8 项数值按汇总规则更新,03 战斗读得到。**可 gdUnit4 测。**
5. **只读掉落包(手动验证):** 悬浮窗能点开面板、列出包内每件信息、确实不能换/不能分解。**手动 Play 验。**
6. **挂机不淹包(手动观察):** 后台跑一段,白装持续被分解、包里只攒蓝/金、零必做操作(守支柱 1)。**手动 Play 验。**

> A-2(a):**04 不验"持续变强爽感"**——那需 05 换装才成立。04 验的是上述机制跑通 + 第一件饰品自动填充那一下。

## 6. 最小版本 / Minimal version
能跑通"掉 → 填空/分解/进包 → 只读查阅"一条线的最小切片:
- 三槽 + 装备数据结构(部位 / ilvl / 基底招牌轴 / 词缀[] / 稀有度)。
- 按部位词缀池 + Tier 表 + ilvl 门槛 roll(**先把生命、暴击率两张表按 §3.4 占位填上跑通,其余词缀表结构就位、数值占位**;数据结构按"任意词缀任意阶"设计,避免扩表重写)。
- 填空优先于分解;自动分解白装→白材料(门槛走 Resource);材料库存 + `material_gained`。
- 只读掉落包面板(列基底+词缀,不可换/不可分解)。
- 背包朴素列表、无上限;穿戴汇总进 03 战士。
- **先不做(归 05/更后):** 手动换装、对比面板、打造/强化消耗材料、自定义分解门槛 UI、套装、宝石、词缀重铸、按词缀的自动优选/替换、满包兜底、跨主题词缀池。

## 7. 与支柱的关系 / Pillar fit
- **支柱 2(变强,权重 3)——主服务对象,但 04 只占"半环"。** A-2(a) 明确:04 把"主动变强"留给 05,自身只做掉落/物品/数据 + 一次填空。**张力(已被用户接受):** 04 单独 playtest 看不到完整变强爽感,这是有意的阶段切分,不是缺陷。
- **支柱 1(伙伴,权重 5)——靠自动分解 + 填空 + 只读包守住。** 零必做操作、洪水自消化、想看才看。**注意克制:** 填空"叮"提示 + 进包微光的强度/频率别变打扰(§4)。
- **支柱 3(惊喜,权重 2)——靠"稀有度=词缀数 + ilvl/Tier 决定数值高低"承载。** "金的!双暴击!ilvl 还高!"的小确幸;白装自动分解后,留包里的本就稀少,每件值得点开看。不靠强随机逼氪。

## 8. Flags / 未决(交下游 + 数值专章)

**F-A · ilvl 来源(交 Explorer/Planner,接线必决):** 装备实例需要一个"物品等级"。02 的 `loot_dropped(kind, rarity)` **不带 ilvl**。需确定来源:
  - 倾向:由掉落来源(打死的怪 / 当前探索进度)给一个整数 ilvl,走配置(如 `EnemyDef` 加 `item_level`,或一个随进度单调上升的全局 ilvl)。**不硬编码。**
  - 这可能要**小幅扩 02→04 的掉落接口**(给掉落带上 ilvl)。Explorer 先摸 02 `loot_dropped` 发射点与是否已有"怪等级/进度"概念,Planner 定接口。**04 落地前必须解决,否则 ilvl 无源、Tier 门槛失去意义。**

**F1 · 数值设计专章(必做后续):** 所有占位待精调——
  - 每条词缀的完整 Tier 表(阶数 / 各阶区间 / 各阶 ilvl 门槛);合格阶的挑选权重(均匀 vs 偏向);
  - 基底招牌轴随 ilvl 的成长曲线;开局全白装的具体数值;
  - 单件分解产材料的数量公式;穿戴多源同维汇总是否需封顶/递减(接 03 已有机制);
