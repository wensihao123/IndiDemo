## 1. 玩家体验 / Fantasy
"我挂着它刷,余光看它自动打。**想知道刷得怎么样了,点一下面板**——左边'我最近的战利品':掉了把金武器,三条词缀带双暴击,ilvl 还挺高,记一笔等回城换;右边'我的战士现在多硬':攻击 12、生命 180、暴击 8%……刚才那条蓝护甲自动戴上时,**生命那行数字当场从 150 跳到 180,绿了一下**——哦,结实了。"

04 自身的爽点 = **「看得见自己在变强、看得见刷到了什么好货」**。它把已经在后台默默发生的变强(自动填空、属性叠加)**显形**,让支柱 2 的"刷→变强"闭环在 v1 里**有据可查**(完整的"主动取舍变强"仍归 05 换装)。**它不新增任何玩法机制,只新增"看"的窗口。**

## 2. 核心循环 / Core loop
1. **挂机产出(自动,后端已落):** 击杀 → 掉落 → 自动填空 / 分解 / 进包。玩家零操作。
2. **想看就看(手动,04 唯一介入):** 悬浮窗一个开关 → 展开**查阅面板**(掉落包 + 当前装备两栏)→ 看「刷到啥」+「我多强」→ 收起,继续挂机。
3. **变强显形(自动反馈):** 自动填空发生时,**当前装备面板对应槽位 + 受影响的属性行高亮跳变**(数字 + 绿色一下)。这是 04 把"变强"从隐性(怪死更快)拉成显性的那一下。
4. **留作回城素材:** 掉落包里的蓝/金件 = 玩家"记下、等 05 回城再挑换"的料。04 只让他**看见并记住**,不让换(换=05)。

> **循环闭合点:** 玩家有动机反复点开面板,因为「数字在涨 / 包里在攒好货」是可见的进度。04 不要求他做任何事,但给了他"查看自己进度"的满足 —— 这正是挂机陪伴(支柱 1)与变强(支柱 2)的交点。

## 3. 关键规则与状态 / Key rules & states

### 3.1 查阅面板:开/关
- 悬浮窗主区(800×250)内一个**查阅开关**(按钮 / 标签页),点开覆盖或挤占战斗区,再点收起回战斗视图。**默认收起**(守支柱 1:不打扰,想看才看)。
- 面板**纯只读**:不能换装、不能手动分解、不能丢弃。任何"动手"操作都属 05。
- 收起态下面板不渲染(同战斗视图"收起即停渲染、模拟照跑"的既有约定)。

### 3.2 左栏:掉落包(读 `PlayerState.bag[]`)
- 列出包内每件(= 蓝/金进包的装备;白件已被自动分解,正常不在包里)。每件一行/一卡显示:
  - **部位**(武器/护甲/饰品)· **稀有度色**(白灰/蓝/金,沿用 `combat_view` 既有 `RARITY_COLOR`)· **ilvl**
  - **招牌轴 + 值**(如"攻击 +5.2 / 攻速 +0.04")
  - **词缀列表**:每条「词缀名 +值」,**可带 T 阶标注**(如"暴击率 +6.3% (T3)");T 阶显示为可选润色,默认显示值即可。
- 空包时显示一句占位("还没攒到好货,白装都自动分解成材料了")。
- **暂不设上限 / 不分页**(沿用旧 C-7;满包兜底见 §7 F-BAG)。

### 3.3 右栏:当前装备 + 8 维属性(读战士 `EquipmentComponent` + `StatsComponent`)
- **三槽穿戴**:武器 / 护甲 / 饰品,各显示当前穿戴件(同左栏的部位/稀有度/ilvl/招牌+词缀);**空槽显式标"空"**(开局饰品槽空,A-1)。
- **8 维属性明细**(用户拍板的"变强看得见"形式):攻击 / 攻速 / 护甲 / 闪避 / 暴击率 / 暴击伤害 / 生命 / 秒回,逐行列**当前合计值**(= 裸职业基础 + Σ装备,`StatsComponent` 已能给出 Final)。
- v1 只填战士一格;面板结构允许日后多角色切换(留扩展位,不在 v1 做)。

### 3.4 自动填空的"变强显形"(§2 步 3 的落点)
- 当 `LootIntake` 判定 EQUIPPED(空槽自动穿)时:**若面板正展开**,对应槽位卡 + 受影响的属性行**高亮跳变**(数字更新 + 绿色闪一下 ~0.3s)。
- **若面板收起**:沿用既有战斗日志一行(已有)+ 一次轻"叮"提示(§4);不强制弹开面板(守支柱 1)。
- 04 阶段这基本只在**第一件饰品掉落填进空项链槽**时触发一次(开局武/甲已满,A-1 后果);设计上接受这"少而珍贵的一下"。

### 3.5 数值粗定(本轮交付"一版大概",精调交 num-smith)
> 当前配置(`data/config/*.json` + `EnemyDef`)的数值是**结构占位**。本文给**design intent(每类数值该带什么手感)**+ 指出现状 gap,**具体数字交 §7 F-NUM 数值设计师**。

**(a) 稀有度掉落频率 —— 有明确 design intent,且现状有 gap 需修:**
- **意图(支柱 3):** 金 = 小确幸,应**稀有**;蓝 = 偶尔;白 = 常见(且多半立刻自动分解)。掉到金应"哦?!",不是家常便饭。
- **现状 gap(读 `combat_arena.gd:164`):** 掉落稀有度目前是**均匀随机**(白/蓝/金等概率),**没有读** `EnemyDef.rarity_weight_white/blue/gold`(普通怪 80/18/2、Boss 50/38/12 这些权重字段**闲置**)。→ 当前金装和白装一样常见,**直接违背支柱 3**。
- **粗定取向:** 让掉落改读 `EnemyDef` 的稀有度权重(字段已就位,值已大致合理);num-smith 再精调权重曲线。**这是接线 + 数值合并的一条线,交 Planner + num-smith。**

**(b) 掉落种类(金币/材料/装备)—— design 取向 + scope 提醒:**
- **现状(读 `combat_arena.gd:163-165`):** 每次掉落**必是装备**(`weight_gold/material/equipment` 字段闲置,kind roll 未接)。
- **取向:** 04 的 fantasy 靠"掉装备"承载,装备掉落**别稀疏到看不见**(若日后按 EnemyDef 的 gold70/mat22/equip8 接,装备只占 8%,前期会"半天不出装",伤 fantasy)。建议 v1 **装备掉落保持相对常见**;金币/材料是否在 04 接成真产出,属 scope 取舍 → 见 §7 F-KIND(倾向:材料已有"白装自动分解"这一来源,金币/直掉材料可留后续)。

**(c) 部位选择:** 现 `combat_arena.gd:163` 均匀随机三部位。**可接受占位**,num-smith 可选给权重(无强 design 诉求)。

**(d) 词缀 Tier 表 / base 曲线 / 合格阶权重 —— 纯数值,全交 num-smith:**
- 现状:`affix_pool.json` 8 条 stat 的 Tier(`max_hp` 10 阶、`crit_chance` 5 阶,其余多为 2 阶 T2/T1)、`item_bases.json` base 曲线、合格阶权重(现全 `weight:1.0` = 均匀)均为占位。
- **design intent(给 num-smith 的手感目标,非数字):**
  - **前期"叮叮叮"热闹**:开局低 ilvl 也能频繁出能穿的白/蓝件,让自动填空的正反馈早出现。
  - **ilvl 拉开纵深**:打更深的怪(更高 `item_level`)→ 解锁更强 Tier → 肉眼能感到"深处的装备明显更猛"。
  - **稀有度=词缀条数**(白0/蓝1-2/金3-4,`loot_tables.json` 已如此),**条数本身就是爽点**:金件三四条词缀"读起来满"。
  - **合格阶权重**:v1 可先均匀;若要"深 ilvl 更易出强阶"的曲线感,留 num-smith,但**别复杂到玩家算不清**(守可读)。
- **分解门槛**:现 `loot_tables.json` = 白全分解、`material_per_decompose:1`。**沿用**(design 上合理:白装洪水自消化);自定义门槛 UI 仍归 05。

### 3.6 失败/边界状态
- **空包**:正常态(白装都分解了),面板给占位文案,不是错误。
- **三槽全满后**:不再触发"变强显形";后续掉落只走分解/进包,面板左栏在攒。这是 04 的稳态,符合预期(持续变强靠 05)。
- **面板展开时战斗继续**:战斗在后台 tick 照跑,收起面板能看到期间的战果(血量/进度已推进)。

## 4. 反馈 / juice 意图 / Feedback & juice intent
> 仅意图,交 Art/Game Feel;不指定素材。沿用 `combat_view` 既有视觉语言(稀有度色、轻 FX)。

- **打开/收起面板**:干脆,无重过场(挂机工具,别仪式感过重)。
- **自动填空"变强显形"(§3.4)**:面板内对应属性行**数字跳变 + 绿色一闪**(↑)、槽位卡轻亮;面板收起时退化为日志行 + 一次轻"叮"。**不弹 OS 通知、不抢焦点**(支柱 1 硬线)。
- **掉落包新增蓝/金件**:左栏角标/微光提示"有新货",不打扰(战斗区已有光柱,不重复轰炸)。
- **属性行的"好"暗示**:8 维数字本身够读;**04 不做"对比已穿戴的绿↑红↓"**——那套对比反馈是 05 换装的核心,04 只回答"我现在多强 / 我刷到了啥",不回答"该不该换"。
- **克制提醒**:面板是"想看才看"的被动信息,所有主动提示(叮/角标)强度要低于战斗本身,别喧宾夺主。

## 5. 成功标准 / Success criteria(可独立验收)
- **手动 Play(支柱体验):**
  1. 能点开/收起查阅面板;面板确实**只读**(无任何换装/分解/丢弃入口)。
  2. 左栏正确列出包内蓝/金件的 部位/ilvl/稀有度色/招牌+词缀;白装不在包(已分解)。
  3. 右栏正确列出三槽穿戴(空槽标"空")+ 8 维当前合计值,数值与战斗表现自洽(穿了护甲血就更厚)。
  4. **第一件饰品自动填空时**,面板内饰品槽 + 生命/闪避/秒回相关行**可见跳变**(若面板展开)。
  5. 挂机一段时间:白装持续被分解、包里只攒蓝/金、零必做操作(支柱 1);金装明显比白装稀有(支柱 3,**前提:稀有度权重已接,见 §3.5a**)。
- **gdUnit4 纯逻辑(数值/掉率,若接了稀有度权重):** 稀有度权重 roll 的分布符合 `EnemyDef` 配置(可 seed 测);现有 generate/intake 测继续绿。
- **数值"粗定"达标线:** 前期能频繁看到能穿的白/蓝件触发填空;打到更深怪能 roll 到肉眼更强的词缀;金件读起来"词缀满"。**精细平衡不在本轮验收**(交 num-smith)。

> **A-2(a) 仍成立但已升级:** 旧版说"04 看不到变强爽感、等 05"。本收窄版**主动把'变强看得见'拉进 04**(右栏 8 维 + 填空显形),让支柱 2 在 v1 里**至少有一半显性兑现**;完整的"主动取舍换装"才留 05。

## 6. 最小版本 / Minimal version
能跑通"挂机 → 点开面板看见刷到啥 + 自己多强 → 自动填空时看见数字涨"的最小切片:
- **一个只读查阅面板**:左栏读 `bag[]` 列表、右栏读战士三槽 + 8 维属性。开/关开关。
- **自动填空显形**:面板展开时对应属性行跳变高亮(收起时退化为既有日志 + 轻提示)。
- **稀有度掉落权重接上**(读 `EnemyDef.rarity_weight_*`),修掉"金白等概率"的 gap —— 这是支柱 3 在 v1 成立的最低要求。
- **数值粗定一版**(num-smith 之前的可玩基线):前期能出能穿的件、ilvl 能拉开强度差。
- **先不做(归 05/更后):** 手动换装、对比面板(绿↑红↓)、打造/强化消耗材料、自定义分解门槛 UI、手动分解/丢弃、金币/直掉材料 kind、满包兜底、多角色面板切换、套装/宝石/重铸、按词缀自动优选替换。

## 7. Flags / 未决

- **F-NUM · 数值精调交 num-smith(必做后续):** 本文只给数值的 **design intent / 手感目标**(§3.5),具体数字(词缀 Tier 区间与门槛、base 曲线、合格阶权重、稀有度权重曲线、分解产出量、各怪 `item_level`/`drop_chance`)留专门的数值设计师 skill 精调。**交接物 = §3.5 的手感目标 + 现有占位配置作起点。** 与 03 的数值专章应合并成一份总表统一精调。
- **F-RARITY-WIRE · 稀有度权重未接(交 Planner,本功能内修):** `combat_arena.gd:164` 用均匀随机稀有度,未读 `EnemyDef.rarity_weight_*`(字段已就位)。→ 现状金装不稀有,违背支柱 3。**这是 04「数值定稿」必须接的一条线**(接线 + 数值,属本功能范围)。建议 Planner 把它纳入 04 的实现 PLAN。
- **F-KIND · 掉落种类是否接(scope,交 Producer 确认):** 现每次掉落必是装备(`weight_gold/material/equipment` 闲置)。04 的 fantasy 靠掉装,**倾向 v1 保持装备掉落常见**;金币/直掉材料是否在 04 接成真产出 = scope 取舍。**倾向:推后**(材料已有"白装自动分解"来源,金币经济归 05/后续)。请 Producer 拍是否纳入 v1。
- **F-BAG · 满包兜底(推后):** 掉落包暂无上限(C-7)。v1 不做满包处理;若 playtest 发现长挂机包爆,再定(自动分解最旧的蓝件?提示?)。
- **F-ARCH-OK · 不需要架构改动(确认):** 本功能纯读已落地的 `bag[]`/`EquipmentComponent`/`StatsComponent` + 接一条已存在字段的掉落权重,**不新增数据形态、不破坏边界/不变量**,无需 `/arch-guard`。表现层读态、不写态,符合 ARCHITECTURE §4 不变量。
- **F-REG · 04 旧 artifact 已 superseded:** 旧 FEATURE-DESIGN/PLAN/CONTEXT-FINDINGS 假设旧 director 结构,已在 HANDOFF 标 superseded;下游**以本文 + 落地代码为准**,勿据旧 PLAN 的 8 步 retrofit 重做后端。
