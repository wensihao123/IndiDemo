# FEATURE-DESIGN — 悬浮窗外壳 (Floating Window Shell)

## 1. The fantasy / 玩家体验
这个窗口**不是一个 UI 面板,而是玩家桌面上的一个"小队伙伴"**。
当玩家在干别的事(写文档、看视频),余光瞥到屏幕最下方,会看到自己的小队还安安静静
地待在那儿、还"活着"——这一瞥带来的是**安心与陪伴**,不是"我有任务要处理"的压力。
它存在感恰到好处:想看时一眼就在,需要专注时能收起来、但从不彻底消失(像个一直在身边
的小宠物)。这直接服务支柱 1——它首先是个伙伴,不是个任务。

## 2. Core loop / 核心循环
外壳本身不是玩法循环,它承载两个"陪伴循环":
- **环境瞥视:** 余光感到底部有动静 → 瞥一眼"小队还在/还活着" → 安心,继续做自己的事。
  (要成立,占位角色必须**一直有轻微的活动**,否则读起来像死图。)
- **收起/唤回:** 要专注时一键收起 → 想看时一键唤回。收起后留一个**小入口(handle)**
  常驻屏幕边缘,保证"伙伴从不彻底消失"。

## 3. Rules & states / 规则与状态
- **位置:** 常驻**主显示器最底部、全屏宽**。主游戏区固定 **800×250 居中**,两侧用占位
  背景图铺满,以适配不同屏宽/分辨率。窗口**坐在 Windows 任务栏之上**:底边贴任务栏上沿、
  **不与任务栏重叠**(占用任务栏以上的可用工作区)。
- **置顶:** **默认 always-on-top 开**(随时可见 = 陪伴);提供一个**快捷键可切换**置顶开/关
  (需要专注时一键让它退到普通层级)。收起态不应遮挡任何东西。
- **窗口状态机:**
  - `展开 (expanded)`:完整显示,占位角色在动。默认开机即此状态。
  - `收起 (collapsed)`:主体隐藏,仅在屏幕边缘留一个小 handle(图标/小条)。
  - 两态之间一键切换;切换是**平滑过渡(滑下/滑上)**,不是瞬切。
- **"会动的占位"(本期的可见成果):** 窗内放一个占位角色 sprite,做**待机微动**
  (呼吸/轻微上下浮动)。它唯一的职责是证明"窗口活着、循环在跑"。
- **边缘状态 / 容错:**
  - 异形分辨率 / 超宽屏:主区始终 800×250 居中不拉伸,两侧背景延展或平铺兜底。
  - 屏幕过窄(<800):主区等比缩放至可见,不被裁切。
  - 窗口定位失败时有兜底位置,**占位动画无论如何都要继续**(永不"假死")。
- 无输赢/失败条件(这是容器,不是玩法)。

## 4. Feedback & juice intent / 反馈与 juice 意图
(交给 Art / Game Feel,我只给意图,不定具体素材)
- **"活着"感**是本期 juice 的全部重点:占位角色的待机微动要自然、低频、不抢眼。
- 收起/唤回的滑动过渡要**轻快顺滑**,像把伙伴轻轻塞进/拉出屏幕边,带一点缓动。
- handle 在收起态可有极轻微的"在那儿"提示(如偶尔的一下小动),但绝不闪烁骚扰。
- 本期音频可选/留空;若有,收起/唤回一个柔和短音即可。

## 5. Success criteria / 成功标准(playtest 时这样判断)
- 把它开着去做别的事 20 分钟——它**有没有打扰你**?(不该有)
- 余光瞥一眼,能不能**立刻感到"小队还活着"**?(占位在动 = 成立)
- 收起→继续工作→想起它→唤回,这套**顺不顺手、烦不烦**?
- 它**有没有挡住你需要用的东西**(任务栏、其它窗口底部)?(关键体验风险,见 Flags)
- 在你自己的显示器分辨率上,布局**有没有错位/被裁**?

## 6. Minimal version / 最小版本
仍能交付"活着的陪伴"幻想的最小切片:
- 固定定位的"底部全宽"窗口 + 800×250 居中主区 + 两侧占位背景。
- **一个**待机微动的占位角色 sprite。
- 收起 / 唤回(含一个常驻 handle)+ 平滑过渡。
- 砍掉:透明窗/点击穿透、拖动、多显示器精调、设置项、音频。
