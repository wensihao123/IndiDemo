# IDEA — 自动战斗循环 (Auto Combat Loop)

> Design Jam 的粗胚。与用户一问一答磨出,刻意留洞(见 §8),交 Game Designer 细化。
> 本 jam 已对齐 project-context 支柱与 BACKLOG(02 含两条 MVP 地基:4 格队伍 slot + 后台持续推进)。

## 1. 一句话 / One-liner
把 1 个战士(4 格队伍只填了第 1 格)丢进一片区域,它就**自动横版打怪**:平时刷波稳定产出、
攒够攻势就推进到更深更硬的一段;窗口在前台你能看它走位挥砍掉装,切到后台它照样替你刷。

## 2. 玩家体验 / Fantasy
"我的小队正在替我卖命。" 玩家不碰战斗,但余光一瞥就知道——队伍活着、在往前啃、在掉东西;
偶尔一道金光说明"刚出好货了"。是**陪伴 + 默默变富的安心感**,不是要你紧张应对的战斗。

## 3. 核心循环(初稿) / Core loop (draft)
**区域内(本功能的循环):**
刷波(怪从右来 → 战士自动迎击 → 掉血/飘字/掉落迸出)→ 稳定产出金/材料/装备事件 →
攻势/进度攒够 → **推进**到更深更硬的一段 → 继续刷(更硬 = 产出更好)→ 回流诱你留下……
**宏观(跨功能,本功能只做产出端):**
02 的产出喂给 03 装备 / 04 回城变强 → 变强后回来啃更硬的段(支柱 2 的"刷→调→挑战")。

## 4. 关键规则与状态 / Key rules & states
- **战斗全自动**,玩家不输入(支柱 1 硬约束)。
- 队伍 = **4 格 slot**,v1 只填第 1 格战士;数据结构从一开始支持 N 人(BACKLOG 地基①)。
- 区域内有"**刷波态**"与"**推进态**";二者切换的闸门待定(见 §8)。
- 怪:2-3 种;有血条、会被击败、会掉落。
- 战士有 HP、会受伤;受伤/团灭如何收场待定(见 §8)。
- **后台持续推进**:窗口收起/失焦/被别的软件挡住时,探索模拟照常跑(BACKLOG 地基②、支柱 1 根基)。
- 产出 = **掉落事件流**(金 / 材料 / 装备),作为接口喂给 03。

## 5. 反馈 / juice 意图 / Feedback & juice intent
- 战斗演出:走位、挥砍、怪受击/被击败、伤害飘字、掉落物从怪身上迸出。
- **掉落按稀有度分级高调**:白 = 默默进包;蓝 = 轻响 + 光柱;金 = 全屏一闪 + 音效 + 约 0.3s 停顿。
- ⚠ **张力(留 Game Designer 调,见 §8)**:全屏闪/停顿在"桌面常驻、你正用别的软件"时可能**打扰**
  (撞支柱 1);高频高调又会**滑向赌场**(撞支柱 3)。金装演出的强度/频率/作用域(窗口内闪
  vs OS 级)需调到"有中奖感但不打扰、不赌场化"。

## 6. 最小版本 / Minimal version(最小值得做原型的切片)
- 1 区域、1 战士、2-3 种怪,**横版自动战斗能看**(走位/挥砍/飘字/掉血)。
- 刷波**产出掉落事件**(先能产出"掉了一件 X 稀有度的东西"这个事件;物品具体词条归 03)。
- **后台模拟跑得起来**:窗口收起/失焦时进度仍推进——哪怕前台演出暂停,数值模拟不停。
- 队伍数据**按 4 格建,只填 1 个战士**。
- 暂不做:推进闸门精细调、团灭惩罚设计、回城衔接(04)、多怪种数值平衡、装备词条池(03)。

## 7. 与游戏支柱的关系 / Pillar fit
- **支柱 1(权重最高)**:全自动战斗 + 后台推进 + 余光可读 = 正中靶心。唯一张力是金装高调演出
  别变成"逼你盯屏 / 打扰你手头的事"。
- **支柱 2**:02 是"刷→调→挑战"里**"刷"的产出引擎**,并用"推进到更硬段"提供够不着的钩子。
- **支柱 3**:掉落分级高调服务"偶尔的惊喜";但要靠 Game Designer 把金装频率压低、别赌场化。

## 8. Open threads / 未决问题(留给 Game Designer 收敛)
- **推进闸门**:"攻足触发推进"到底靠什么——击杀数 / 进度条 / 清波阈值 / 计时?推进是**无限深**
  还是区域有**底 / 小 boss**?(决定里程碑感与节奏曲线"前期快迭代→后期稳态"怎么落地)
- **玩家在 02 里有没有杠杆**:纯观众,还是有最小操作(开始探索 / 选区域 / 主动休整)?这条与 04
  城镇边界相关——建议先想清 02/04 各管什么。
- **战士受伤 / 团灭怎么收场**:会死吗?死了自动撤退 / 原地复活 / 进度回退?(直接影响"低负担陪伴"手感)
- **前台演出 ↔ 后台模拟如何对齐**:后台只跑数值模拟;回到前台是"补演最近发生的"还是直接显示当前态?
  (有演出的横版战斗在窗口不可见时无法渲染,模拟与表现必须解耦)
- **金装演出**:强度 / 频率 / 作用域(窗口内闪 vs OS 级通知),怎样既有中奖感又不打扰、不赌场化?
- **掉落→03 的接口边界**:02 只产"掉了一件 X 稀有度的东西"事件、细节归 03?(建议如此,避免 02 越界)
- 〔**Producer 范围 flag**〕"有演出的横版战斗"(怪精灵 + 攻/受击表现 + 飘字 + 掉落迸出)成本远超 01
  的 3 张占位图。本 jam 用户选了"有演出",但 **v1 是否就上演出版、还是先符号/日志版留钩子,建议回
  Producer + Art Spec 评估**后再让 Game Designer 定稿。
