# FEATURE-DESIGN — 自动战斗循环 (Auto Combat Loop)

> 承 IDEA.md;Producer 已裁定 **v1 = 符号/轻演出版**(复用 01 微动战士 + 战斗符号/日志 +
> 分级掉落光效;横版全套演出在 Later)。本设计建在该天花板上。IDEA §8 的 7 条 Open threads
> 已逐条收敛(见 §7 标注),用户已拍板关卡/团灭/修整三大块。

## 1. The fantasy / 玩家体验
"我的小队在替我**稳步攻关**——我只需偶尔抬头决定'再推一关,还是回城整备',它就自己打下去。
打不过就退回去刷,刷强了再上。" 战斗不要我操作;参与感来自**关口处的取舍**(推进 vs 修整)和
**看着进度一关关往前**的踏实,以及偶尔一道金光的小确幸。是陪伴 + 稳步变强,不是紧张应对。

## 2. Core loop / 核心循环
**关内(自动):** 第1场景 → 第2场景 → 第3场景(小怪难度逐级递增)→ 关底 Boss。每个场景自动清怪后
自动进下一个,**无需玩家操作**。
**关口(玩家取舍):** 击杀 Boss → **永久解锁下一关** → 5 秒倒计时(默认自动推进到下一关第一场景)→
玩家可在倒计时内点**修整**回城整备(花战利品变强,= 04)。
**卡关回流:** 团灭 → 退回上一个普通场景**无尽刷怪**(持续产出、攒等级/装备)→ 同时出现
**推进**与**修整**按钮 → 玩家觉得够强了点推进重试下一关,或点修整回城。
**宏观(跨功能):** 刷出的金/材料/装备 → 喂 03 装备 / 04 回城变强 → 回来啃更硬的关。
→ 重复的理由:下一关的 Boss 是"够不着的怪"(支柱 2);变强够到它,就解锁更深的关与更好的产出。

## 3. Rules & states / 规则与状态
**队伍与存活:**
- 队伍 = 4 格 slot,v1 只填第 1 格战士(数据结构支持 N 人)。
- 战斗中只要队伍**≥1 人存活**则战斗继续;**全员倒下 = 团灭**。

**关卡结构:**
- 1 关 = **3 个普通场景 + 1 个关底 Boss**。普通场景小怪难度逐级递增。
- **Boss 是一次性关卡门**:击杀一次即**永久解锁下一关**,此后永不需要重打该 Boss。

**团灭与回退(用户拍板,务必照此):**
1. 团灭 → 记录"在第 N 关团灭" → 回退到**上一个普通场景**做**无尽刷怪**(怪无限来,持续产出)。
2. **Boss 永不作为刷怪/回退目标。** 若团灭发生在某关第一场景(其时序上的"上一场景"是上一关 Boss),
   回退**跳过 Boss**,退到**上一关第三场景**刷;此时点推进 → 进入下一关第一场景(不重打 Boss)。
3. 调值意图(非硬规则,走配置/Resource,见 §7):**关底 Boss 略强于下一关前两个场景**,使"打得过
   Boss 就大概率推得动下一关前段",避免频繁回退;也避免"卡在某关第一场景→反复重打上一关 Boss"。

**队伍回血 / 难度模型(playtest 后拍板,2026-06-18,承 REVIEW S2):**
- **何时回满:** 全队回满血发生在四个时刻——① 每清完一个普通场景过场时;② 通关 Boss 进下一关时;
  ③ 卡关刷怪**刷满一轮**(= 杀够该场景 `kill_count`)时;④ 团灭回退落点(原有)。
- **设计意图(为什么这么定):** 移除"跨场景的血量持续损耗(attrition)"。消耗战会逼玩家**频繁盯屏、
  频繁回城补给**,直接违背支柱 1"低负担陪伴、安心刷"。挂机伙伴该有的体感是"它替我稳稳打下去",
  而不是"我不看着它就会因为血空被拖死回退"。playtest 实测旧的越刷血越低 → 连环回退,正是这种挫败。
- **难度门槛因此明确改挂两处(替代 attrition):**
  1. **单场景内**:一个场景从满血开打,必须能在 `kill_count` 次击杀内**扛得住**;调不动则该场景本身即门槛。
  2. **Boss**:满血进 Boss,Boss 是关口主门槛(承"Boss 略强于下一关前两场景"的调值意图)。
- **⚠ 平衡硬约束(交 Planner/playtest,见 §5/F1):** 回满**不等于无敌**。卡关必须**真能发生**——
  即存在"满血也打不过这个场景 / Boss"的配置点,否则取舍循环(推进 vs 修整)失去意义、支柱 2 落空。
  调值时要保证至少 Boss(及关2 起的部分场景)能在满血下仍构成门槛。

**推进 / 修整 按钮(玩家仅有的杠杆,战斗仍全自动):**
- **通关 Boss 后:** 5 秒倒计时,期间显示**修整**按钮;**无操作 → 自动推进**到下一关第一场景。
  (默认前进=伙伴自己打下去;想花战利品就点修整。)
- **卡关刷怪时:** **推进 + 修整 按钮同时出现**。点击后,对应操作在**本轮刷怪结束后**执行:
  - 推进 = 重新尝试下一关(从回退点向前的目标关第一场景)。
  - 修整 = 回城整备(目的地 = 04 城镇;02 只提供"离开本轮、去整备"的钩子)。
- 关内场景推进(1→2→3→Boss)自动,无按钮。

**产出:** 战斗(含无尽刷怪)持续产出**掉落事件流**(金 / 材料 / 装备),作为接口喂给 03。

**后台持续推进:** 战斗是一个**持续运行的模拟**,窗口收起/失焦/被遮挡时照常 tick;前台画面是
当前模拟状态的**读出**,回到前台**直接显示当前态**(不补演错过的战斗)。

**边缘态:** 第 1 关第 1 场景团灭(无上一关可退)→ 原地无尽刷该场景,不回退到不存在的关。

## 4. Feedback & juice intent / 反馈意图(交 Art/Audio/Game Feel;符号/轻演出版)
- **战斗(轻演出):** 复用 01 微动占位战士;怪用简单符号/占位精灵从右侧出现;命中/击败用
  伤害飘字 + 战斗日志行("⚔ 击败 哥布林 +3 金");场景推进/进度用一条进度读出("第 N 关 · 场景 2/3")。
- **掉落分级高调(承 IDEA §5):** 白=默默进包;蓝=轻响 + 光柱;金=**窗口内**一闪 + 音效 + 极短停顿。
  ⚠ 守支柱:金装演出限**窗口内**、**不弹 OS 通知/不抢焦点**,频率压低,避免"打扰你手头的事"(支柱1)
  与"赌场上瘾感"(支柱3)。
- **关口反馈:** Boss 击杀给一个明确的"通关"小庆祝 + 倒计时可见;团灭给一个**柔和、非惩罚**的"撤退到
  上一场景,继续变强"提示(要"安心刷"不要"挫败")。
- **可读性:** 余光一瞥就能看懂"打到第几关第几场景、在推进还是在卡关刷"(支柱1 核心验收点)。

## 5. Success criteria / 成败判断(playtest 问题)
- 玩家是否会自然地在"卡关 → 刷 → 推进"间循环,而**不觉得被迫盯屏**?(支柱1)
- 团灭 → 回退 → 无尽刷,体感是"**安心地变强**"还是"被惩罚/挫败"?(若偏挫败,需调柔反馈/回退规则)
- 推进 vs 修整 的取舍时刻,是否带来"**该不该再赌一把/还是先整备**"的主动参与感?(支柱2)
- 挂后台一段时间回来,能否**一眼看懂进度**(第几关第几场景)?
- 金装掉落是"惊喜"还是"打扰/焦虑"?窗口内演出强度是否合适?(支柱3)
- "Boss 略强于下一关前两场景"的调值是否真能避免频繁回退/卡死循环?
- **(回血模型,承 S2)** 过场景/满轮回满后,卡关是否仍**真的会发生**?会不会因为"满血开打"把难度门
  抹平到几乎打不死、循环失去张力?反之,单场景/Boss 是否反而变成"一面墙"硬卡(回满也过不去、毫无渐进感)?
  ——理想是"满血能稳推大多数场景,但 Boss / 关2 起的硬场景需要回城变强才过"。

## 6. Minimal version / 最小版本(仍交付幻想,守符号版天花板)
- **2 关**:关1(3 普通场景 + Boss)+ 关2(至少第 1 场景,且调得稍硬以便能触发卡关)。两关才能演示
  "Boss 门 + 解锁推进 + 团灭回退 + 卡关刷"的完整闭环。
- 队伍按 **4 格建,只填 1 个战士**;"≥1 人存活则继续 / 全灭团灭"规则到位(已为 N 人留地基)。
- 自动战斗(符号/轻演出)+ 逐场景推进 + Boss 一次性门 + 团灭回退无尽刷。
- **推进按钮** + 团灭回退/卡关刷 闭环。
- **掉落事件流**(产出"掉了一件 X 稀有度的东西 / 金 / 材料"事件;物品词条细节归 03)。
- **后台模拟**跑得起来(窗口收起/失焦时进度仍 tick)。
- **修整按钮可present 但目的地是 stub**(真正回城整备 = 04;02 minimal 里点修整可先到占位/暂停态)。
- 暂不做:横版全套演出、多职业/组队、技能、多关数值精调、词条池(03)、真实城镇(04)、离线结算。

- ✅ 前台演出↔后台模拟 = **模拟持续 tick,前台为当前态读出,不补演**(设计师定;实现细节交 Planner)。
- ✅ 金装演出度 = **窗口内演出、不弹 OS 通知、频率压低**(设计师定;支柱3 调味)。
- ✅ 掉落→03 接口 = **02 只产"掉了 X 稀有度一件 / 金 / 材料"事件,物品细节归 03**(设计师定)。
