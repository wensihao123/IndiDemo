# ASSET-SPEC — 城镇枢纽 + 五子板块视觉(全局 UI·juice 轮)

> **来源蓝图 = UX-CHANGE-04**(城镇枢纽 + 五板块 IA)+ 用户 6 张参考图。本 spec 把那份**布局蓝图**
> 落成**具体素材清单**:每件的尺寸 / 锚点 / 九宫格切边 / 命名 / 格式 / palette 约束 + prompt brief +
> 客观验收。画风/配色/线条遵 **STYLE-BIBLE 方案 A 暖木治愈** + §8 UI 系统;工程纪律遵 `style-basic-2d.md`。
>
> **范围(2026-06-20 用户拍板「整轮五板块+枢纽」)**:A 组 共用 UI 框架件(一次出、五板块复用)·
> B 组 图标集(32×32)· C 组 城镇枢纽场景(背景 + 4 建筑入口)· D 组 板块专属图(立绘框/出征场景/酒馆场景)。
> **只做 UX-CHANGE-04 的 A 层(映射现有系统)**;B 层无系统支撑项(货币条/收益秒/每日任务·活动·成就/
> 制作/技能管理/消耗·任务/成功率/自动战斗开关)= **不出素材**(硬 NO:不为没影系统提前抽象)。
>
> **关键画幅前提(SMM 已裁决)**:城镇复用 **800×250 贴底全宽窗**(非放大窗,详 STATE-MACHINES.md §6 #8)。
> 故所有板块在 **800×250 逻辑画布**内排版——比参考图(竖幅更高)**更扁更紧**,框架件用**九宫格(NinePatch)
> 适应窄高**,信息密度按"余光可读(支柱 1)"取舍,不照搬参考的满屏信息。

## 0. 一个需先拍的画风决策(框架件:flat StyleBox → 九宫格)

参考图的面板是**手绘暖木 + 金色花边的厚重框**;而 STYLE-BIBLE §8.2 现锁的是 **flat `StyleBoxFlat`**
(纯色圆角 + 2px 边)——那是**占位期**基线。本轮 = 全局 UI·juice「占位→上皮」轮,**目标观感 = 参考图的
暖木花边框**。

**取向(用户 2026-06-20 已确认花边)**:出一套**小而可复用的九宫格框架件**(A 组),五板块共用;
**不为每板块单独画框**。STYLE-BIBLE §8.2 的 StyleBox 数值(圆角/间距/字号阶梯)**仍是内距与排版的事实源**,
九宫格只接管"框的皮"。已据此更新 STYLE-BIBLE **§8.6**(见该文件)。
> 纯 flat StyleBox 留作可无缝降级的退路,本轮不取。用户先按花边出图,效果好再按需回头细调 STYLE-BIBLE。

## 2. Naming & format(命名与格式 —— 先锁,勿后改)
- 文件名全小写 `snake_case`(见 §1),扩展名 `.png`。
- 目录:框架件 `assets/sprites/ui/frame/`、图标 `assets/sprites/ui/icons/`、城镇场景与建筑/板块专属
  `assets/sprites/town/`。(沿用 02 既定 `enemies/`、`hero/`、`bg/`、`ui/` 习惯,新增 `ui/frame`、`ui/icons`、`town`。)
- 格式 **PNG**;除 `bg_town`(满底不透明)外尽量透明;`stage_preview_*`/`bg_tavern` 可不透明(嵌在 `frame_inset` 里)。
- 九宫格件须在 §1 给定 `patch_margin` 下**中心与四边平铺无缝**(角可花边、边不可有方向性图案致接缝)。
- 源文件(.kra/.psd)放 `res://_source/`(带 `.gdignore`,不导入/不进导出包)。
- **命名一旦被场景/Theme 引用就别改**(改名级联断引用,style-basic-2d §5.2)。

## 3. Style constraints(外观约束 —— 全部回扣 STYLE-BIBLE)
- **画风**:平滑手绘、**非像素**;圆润、暖调、清晰描边、轻盈治愈(§0/§3)。九宫格框 = **暖木 + 金色细花边**
  (呼应参考图),但**克制**——服务支柱 1「余光可读、不抢注意」,花边不可喧宾夺主、不可降低文字对比。
- **描边**:统一 `#3a2e2a` 暖棕(非纯黑);图标主轮廓 ≈2px@32 画布(§8.4);框边 2–3px。
- **palette**(§4 方案 A,**新素材一律落这套,偏色即不通过**):
  - 框/面板:底 深紫灰 `#2b2536`(半透),边/花 暖木 `#c98a5e` + 暖橙 `#e8943c` 点睛;内嵌区更深。
  - 按钮:`btn_normal` 暖木 `#c98a5e` 底 / 奶白 `#f2e9dc` 文字;`btn_primary` 暖橙 `#e8943c` 底(主操作)。
  - 页签:`tab_on` 暖木提亮 / `tab_off` 去饱压暗。
  - 选中/高亮:暖橙 `#e8943c`(`slot_frame_sel`、`list_row_sel`、建筑悬停)。
  - 城镇场景:暖木建筑 + 清新绿 `#7fb069` 草木 + 深绿 `#4a7c59` 远景 + 天光奶白 `#f5e9d6`(暖调治愈、低对比)。
  - 图标内填:取主 palette;材料/属性图标可借少量饱和点睛,但描边/基调统一。
- **稀有度色锁死**(勿改):白 `#d9d9d9` / 蓝 `#6699ff` / 金 `#ffd24a`(已硬用于掉落 FX `RARITY_COLOR` + §8.5)。
  槽位稀有度边框**运行时染**这三色,**不烘焙进 `slot_frame`**。
- **可读性(支柱 1 核心验收)**:800×250 内信息密;图标在 16–24px 显示仍要一眼分清;
  框花边不得压低文字/数值对比;选中态余光可辨。
- **不得包含(运行时状态/文字,硬红线)**:任何文字/数字(标题/按钮字/Lv/数量/价格/属性值)、血条、
  选中框(框选态另出 `_sel` 图,但不叠数字)、稀有度色(运行时染)、禁用/hover/pressed/focus 态
  (全运行时 `modulate`/描边,§8.3)。**九宫格件不得画进会随拉伸变形的方向性文字/图案**。
