# FEATURE-DESIGN — 05 城镇:手动换装 + 对比面板 + 装备强化

> 本功能是**支柱 2「变强够到下一个怪」在 v1 的高光落点**:挂机产出(刷)在这里被玩家**主动转化**成战力(调),
> 再带出去挑战更硬的怪(再挑战)。后端持久层(roster / 背包 / 材料 / `EquipmentComponent` 无损穿脱)已由
> `00-foundation-redesign` 落地;05 = **点亮"回城主动构筑"这一刻** + 引入 v1 第一个材料消耗口(强化)。

## 1. The fantasy（玩家该感到什么）
玩家在挂机里攒下一堆掉落和材料,**回到城镇这个属于自己的、安静的工作台**,亲手把队伍调得更强:
- 「**这件比我现在穿的好吗?**」—— 一眼看清逐项差值,做出**自己的**取舍,而不是被系统自动决定;
- 「**我把这把武器养起来了。**」—— 花攒下的材料把心爱的装备一级级强化,看着数字稳稳往上走;
- 出城时带着「我刚刚变强了,去看看能不能啃下那个怪」的**明确动机**。

核心情绪:**主动掌控感 + 确定的成长回报**。回城不是负担,是「捣鼓自己 build」的小确幸时刻(守支柱 1 的陪伴基调:
opt-in、玩家自己开、随时关掉回到挂机)。

## 2. Core loop（动作 → 结果 → 反馈 → 再来一次的理由）
```
挂机刷出 蓝/金 装 + 材料(白装自动分解来)
   → 玩家主动「进城」(当前挂机暂停,进入独立城镇界面)
       → ① 换装:从背包挑装备,对比面板看「装上它 vs 当前」逐项差值(绿↑/红↓)→ 穿上更优的,旧的回包
       → ② 强化:选一件装备,花材料 → 强化等级 +1 → 该装备数值上升(确定性,无失败)
   → 「出城」恢复挂机,带着更高战力 → 够到/打过更硬的怪 → 掉更好的装 + 更多材料
   → 想再调,再回城  ↻
```
**闭环成立的关键**:换装与强化都让「下一次挂机能走得更深」,而更深 = 更高 ilvl 掉落 + 更多材料 = 下一轮回城有更多可调。

## 3. Rules & states（具体规则 / 边界态）
**进城 / 出城**
- 玩家从悬浮窗主区有一个明确的「进城」入口;点开 → **当前挂机推进暂停**,切到城镇独立界面(800×250 内)。
- 「出城」→ 关闭城镇界面,**挂机从暂停处恢复**(不重置进度、不结算掉落、不损失)。
- 城镇是**纯主动界面**:不开就完全不影响挂机陪伴;开着时玩家在主动操作,暂停挂机是为「捣鼓时不漏刷、不焦虑」。

**换装(manual equip swap)**
- 作用对象:当前队伍的战士(v1 只 1 人)的 **3 槽(武器 / 护甲 / 饰品)**。
- 从**背包**选一件装备 → 装到其**对应槽**;若该槽已有装备,被换下的那件**无损回到背包**(守不变量 i1 装备无损还原)。
- 可把已穿戴的装备**主动换下**(留空槽或换别件)。
- ⚠ **与「自动填空只增不替」(i3) 的关系**:i3 约束的是**挂机时的自动穿戴**(空槽才自动穿、绝不替换);
  **手动换装是玩家主动行为,允许替换**。两条路径并存不冲突 —— 出城后挂机仍只对**空槽**自动填,不会覆盖玩家手动的选择。

**对比面板(comparison)**
- 在背包里选中/聚焦一件装备 → 即时显示「**装上它** 对比 **当前该槽装备**」的 **8 维逐项差值**:升的绿↑、降的红↓、无变化淡显或省略。
- 差值口径 = 候选件带来的最终属性 − 当前件带来的最终属性(基底 + 词缀 + 强化加成都计入);空槽则与「裸槽 0」比。
- 目的:让「哪件更优」**一眼可辨**,把取舍决策权交给玩家(支柱 2 的主动构筑)。

**强化(enhance / +level)**
- 作用对象:**任意一件装备(三槽通吃,背包或已穿戴均可)**。
- 花费**材料** → 该装备**强化等级 +1** → 其数值获得提升(具体加成幅度 / 成本曲线 / 是否设上限 → **交 num-smith 定**)。
- **设计意图(GD 守的红线,数值之上)**:强化必须**确定性、无失败、纯增益**——花了材料就一定 +1、数值一定涨,
  **绝不做"强化失败 / 掉级 / 碎裂"**(守支柱 3「偶尔惊喜而非赌场」:成长是稳的回报,不是老虎机)。
- 强化加成随装备走:换下来放包里、再穿上,强化等级与加成保留(随 `ItemInstance` 持久)。

**边界 / 失败态**
- 材料不足 → 强化入口禁用 + 明确提示「材料不够」,不报错、不扣半截。
- 背包空 / 槽位空 → 面板显空态(「(空)」),换装/对比照常工作(对比空槽 = 与 0 比)。
- 若强化设了上限(num-smith 定)→ 满级件显「已满级」、强化禁用。
- 出城瞬间挂机恢复:进城前的战斗中间态(血量 / 进度倒计时)按暂停—恢复处理,不因换装重置(具体由 Planner/架构定怎么落,GD 只要求「不惩罚玩家回城」)。

## 4. Feedback & juice intent（交给 Art/Audio/Game Feel 的意图,不指定 asset）
- **换装成功**:对应槽数值**跳变 + 绿闪**(复用 04 已建立的「变强显形」视觉语言,保持一致)。
- **对比差值**:绿↑ / 红↓ 即时、显眼,数字对齐易扫读;这是本功能最高频的反馈,要"扫一眼就懂"。
- **强化成功**:装备数值跳 + 一下**强化特效 + 音**(给"我养强了它"的满足感);材料计数可见地**减下去**(花费有重量感)。
- **材料不足 / 满级**:克制的禁用态 + 一句提示,**不惩罚、不闪红报警**(守陪伴基调,不制造焦虑)。
- 整体调性:城镇是**安静的工作台**,反馈温和确定,不要赌场式的大开大合(对照支柱 3)。

## 5. Success criteria（怎么判断好玩 / 成立 —— playtest 问题）
- 玩家回城后是否**自然地想去比一比、挑一挑**,而不是无脑穿最新掉落?(对比面板有没有制造"取舍乐趣")
- 「装上它 vs 当前」的差值是否**一眼可辨更优**,玩家能不能不算账就做决定?
- 强化是否给出**"我变强了"的即时、确定满足**,而不像在拉老虎机或被成本劝退?
- 「回城 → 调 → 出城 → 够到更硬的怪」这条闭环,前期是否**真的会被反复触发**(支柱 2 成立的信号)?
- 进/出城是否**顺滑不打断陪伴**:开城镇不焦虑(挂机暂停了)、出城即恢复、没有"回城惩罚"?

## 6. Minimal version（仍兑现 fantasy 的最小切片）
- **独立城镇界面**(进城暂停挂机 / 出城恢复),容纳下面三件:
  1. **1 战士 3 槽手动换装**(背包 ↔ 槽,无损,允许主动替换);
  2. **对比面板**(选中背包件 → 8 维逐项差值 绿↑红↓);
  3. **装备强化**(确定性 +1、作用三槽、花材料、数值随等级升;上限/成本/幅度由 num-smith 定)。
- **材料来源维持现状**(白装挂机自动分解,BALANCE §4),强化是 v1 第一个**消耗口**。
- **明确不含(推后)**:招募 / 多角色实际换人、技能升级、其它打造线(合成 / 重铸 / 镶嵌)、**手动分解蓝金补材料**、
  批量强化、套装、强化失败或降级机制、装备出售 / 金币消耗口。

## 7. Flags / Open questions
- **🔴 F-ARCH(阻塞前置,必须先审)** — 「强化等级」= 给可序列化的 `ItemInstance` **新增字段 + 其对 `StatsComponent`
  的加成来源**(存档单元的数据模型变更);且城镇要在**战斗外**读写 roster / 装备 / 可能读模板曲线 →
  `DataRegistry` 由「单消费者(战斗座)」变**多消费者**(ARCHITECTURE.md §6 已预埋此复审点)。
  **用户已明示「牵扯结构修改则先给 arch-guard 审计」。** → **下一步先 `/arch-guard 05-town-gear-upgrade`**
  (定:① `ItemInstance` 强化字段 + 加成如何经 modifier 接入不破 i1/i2;② `DataRegistry` 座位升 autoload vs 注入;
  ③ 暂停—恢复挂机的状态归属)。Planner 在架构落地后才好排计划。
- **🟡 F-NUM(交 num-smith)** — 强化数值全套:每级加成幅度(按槽 / 按招牌轴怎么涨)、材料成本曲线(是否随等级递增)、
  **是否设强化上限**、三槽是否同一强化公式。配套**材料经济校验**:唯一来源仍是白装自动分解,强化作为新消耗口,
  v1 产出 vs 消耗是否平衡(会不会材料荒 / 材料爆)。GD 红线给死:**确定性、无失败、纯增益**;数字在此约束内定。
- **🟡 材料来源是否够(开放,倾向不扩)** — 强化吃材料,但 v1 材料只来自白装自动分解。若 playtest 发现材料供不上,
  备选是「城镇手动分解蓝/金换材料」。**v1 倾向不做**(守最小 + 避免与"蓝金是 build 产出"打架);先记为 num-smith 经济校验的输入,真荒了再交 Producer 拍。
- **🟢 支柱 1 相容性(已判定相容,记录)** — 城镇为独立界面 + 暂停挂机,看似与「后台持续推进」张力,实则相容:
  后台推进约束的是**窗口最小化/失焦时**照跑;城镇是**玩家主动打开**的前台操作界面,暂停是为"捣鼓时不漏刷"。
  二者不冲突。**待 playtest 确认进/出城不破坏陪伴节奏**。
- **🟢 i3 澄清(交 BALANCE / Implementer)** — 需在落地时明确两条穿戴路径:**手动换装可替换**(玩家主动) vs
  **自动填空只增不替**(挂机自动,i3)。建议 num-smith 在 BALANCE 补一句区分,避免实现时把 i3 误用到手动路径。
- **🟢 Producer 知会** — 05 范围按用户拍板**放宽**:强化从 BACKLOG 的「+1 武器」扩到「**所有装备**」。属合理放宽
  (三槽一致体验更完整),但 Producer 应在 BACKLOG 追认这条 scope 变更。
