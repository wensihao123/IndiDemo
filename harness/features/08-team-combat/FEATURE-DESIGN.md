# FEATURE-DESIGN — 08 团战:一波多敌(近战门控 + 远程隔位)

> 本功能把战斗从「单怪车轮」升成「**一波杂兵团**」——挂机画面里同时出现一小撮敌人,有近有远、有前有后。
> 它服务**支柱 1「瞥一眼就懂的陪伴」的画面厚度**(战斗看起来更像回事、更有"在打一群")与**支柱 2 的压力来源**
> (更硬的一波 = 下一个要够到的坎)。地基已预留多实体座位(`CombatArena.enemies: Array[Entity]`、
> `AICombatComponent.in_range` 占位恒真、`select_target` 集火最前),08 = **点亮这些座位** + 引入「近/远站位」这层薄战术。

## 1. The fantasy（玩家该感到什么）
玩家瞥一眼悬浮窗,看到自己的战士**面前堵着一小群杂兵**:前排几个近战凑上来轮流砍,后排还有放冷箭的远程在隔着人输出。
- 「**这波人多,有点顶。**」—— 不是一对一磨,而是被一**团**压着,战斗画面有了"群架感"。
- 「**前面的清掉,后面的才轮得上。**」—— 近战要排队凑近才打得到(车轮),清掉前排是肉眼可见的推进。
- 「**那个远程一直在偷我血。**」—— 后排远程不靠前也能输出,给"先解决谁"留了一点点直觉判断(但仍全自动,玩家不操作)。

核心情绪:**被一群围攻的张力 + 逐个清空的推进感**。全程仍是**全自动挂机**——玩家不点不拖,只是"看着更有料、更有压力"
(守支柱 1:瞥一眼看懂、不增加 APM)。

## 2. Core loop（动作 → 结果 → 反馈 → 再来一次的理由）
```
挂机进入一个场景 → 刷出「一波」敌人(前排近战 N 个 + 后排远程 M 个,同屏)
   → 战士自动集火最前存活敌(沿用 select_target 语义),逐个砍倒
       → 前排近战:只有"凑到射程内"的才打得到战士 → 清掉一个,下一个补上(门控 = 车轮战)
       → 后排远程:从后排位置就能对战士输出(隔位),不受前排门控
   → 整波清空 → 计入清场进度 → 下一波 / 下一场景(更硬的一波)
   → 更硬的波 = 更需要前面攒的战力(换装/强化)→ 回到支柱 2 闭环  ↻
```
**闭环成立的关键**:一波的"硬度"(人数 + 近远配比 + 数值)成为新的难度旋钮,让"变强够到下一个坎"有了更立体的坎,
而不只是单怪 HP 往上堆。

## 3. Rules & states（具体规则 / 边界态）
**一波多敌(team / wave)**
- 一个场景的一场战斗,敌方不再是 1 只,而是**一小撮**(v1 量级建议 2–4,具体由 num-smith 定),**同屏并存**。
- 沿用现有结算:战士每次出手 `select_target` 选**敌方最前存活者**(数组顺序 = 排位),集火逐个清空;清空整波 = 这场清完。
- **不变量 #7 守死**:站位是**抽象的"前排/后排 × 序"**(数组顺序 + 站位类别),**不做真 2D 移动 / 碰撞 / 抛射物**。
  画面上敌人是排开的色块/贴图,逻辑上是有序数组 + 一个"近/远"标签,**没有真实坐标推进**。

**近/远 站位(positioning)**
- 每种敌人带一个**站位类别 ∈ {近战, 远程}**(给 `EnemyDef` 新增字段;近战默认,远程显式标)。
- **近战门控(melee gating)**:近战敌**只有"在射程内"才能打到战士**。射程 = 复用 `AICombatComponent.in_range` 这个
  现在恒真的占位座位,把它**点亮成真判定**:同一时刻只有最靠前的若干近战"够得着"战士、参与出手,其余在后面**排队等位**
  (前面的死了才轮到)。→ 这就是**车轮战**手感:人多但不是所有人同时砍你。
- **远程隔位(ranged reach-over)**:远程敌**不受门控**,从后排位置即可对战士输出(`in_range` 对远程恒真)。
  → 远程是"持续漏的血",制造"该不该先想办法够到后排"的张力(v1 战士够不到后排是**设计内的**,见下)。
- **战士侧(玩家)**:v1 单战士,**单体攻击**——每次出手只打一个目标(集火最前)。**AoE / 群伤明确推后**
  (留给未来技能 / v2 的法师类角色,见 §6 与 F-AOE)。所以面对一波,战士是"一个一个清",远程要等前排清完才轮到被集火。

**清场 / 推进**
- 现有 `SceneConfig.kill_count`(清场所需击杀数)语义保留;一波多敌下,**清空整波**累加进清场进度
  (一波 K 只 = 一次发生 K 次 `enemy_defeated`)。波与场景清场的具体关系(一波即一场 / 多波一场)→ 交 Planner+num-smith
  落,GD 只要求"杀干净一波是肉眼可见的一段推进"。
- **团灭**:战士被这波打死 → 沿用现有 `party_wiped` / `retreat_after_wipe`,不新增惩罚。

**边界 / 失败态**
- 一波里**全是远程(无近战前排)**:门控空置,所有远程直接输出战士(合法,只是更"脆"的一波);属配置自由度,不报错。
- 一波里**全是近战**:无远程漏血,纯车轮;门控决定同时几个能打到 → num-smith 定"门控容量"(同时几个近战在射程)。
- **战士打不到后排远程**:v1 **接受**——战士单体集火最前,前排没清完就够不到后排远程,这是"远程持续压力"的来源,
  **不是 bug**。若 playtest 发现远程过于无解(磨死战士)→ 调远程数值/配比,而非给战士加 AoE(守 F-AOE 推后)。

## 4. Feedback & juice intent（交给 Art/Audio 的意图,不指定 asset · 推后到统一 UI 轮）
> 用户已拍板:**UI/juice 表现层推迟到 v1 功能全做完后统一一轮**。本节只记**意图**,本功能阶段用占位程序美术(色块/Label)验功能。
- **"一群"的呈现**:一波多敌要**一眼看出"在打一群"**——敌人横向排开,前排近战靠前、后排远程靠后,数量感清晰。
- **集火焦点**:当前被集火的最前敌应**可辨**(战士的攻击落在谁身上一眼可见),清掉一个、焦点前移要顺滑。
- **远程漏血**:远程的命中应有别于近战(方向/特效区分),让"血怎么还在掉"对应到"哦是后排那个"——但**克制**,不喧宾夺主。
- **车轮补位**:前排死、后排近战补上的"补位"动作应可读(让门控= 车轮战的手感被看见)。
- 整体调性沿用战斗既有视觉语言(04/02 建立),团战是其**密度升级**,不是另起一套。

## 5. Success criteria（怎么判断好玩 / 成立 —— playtest 问题）
- 瞥一眼能不能立刻看出"**这是在打一群、有前有后**",而不是一堆数字糊在一起?(支柱 1 可读性)
- 近战门控有没有给出**车轮战手感**(人多但逐个轮、清前排有推进感),而不是"一拥而上瞬间被秒"或"和单怪没区别"?
- 远程隔位有没有制造**恰到好处的额外压力**(持续漏血、让人想"先清前排够到后排"),而不是无解磨死 / 或存在感为零?
- 整波的"硬度"是否成为**有效的难度旋钮**——更硬的一波让"前面攒的战力"被真正用上(支柱 2)?
- 全程**仍是全自动、不增加操作负担**吗?(团战不能偷偷把游戏变成要玩家点的 RTS)

## 6. Minimal version（仍兑现 fantasy 的最小切片）
- **敌方一波多敌**:一场战斗同屏 **2–4 只**(num-smith 定),含**前排近战 + 后排远程**两类站位。
- **近战门控**:点亮 `in_range` 为真判定 + "门控容量"(同时几个近战够得着战士),其余排队;前排死则补位(车轮)。
- **远程隔位**:远程 `in_range` 恒真,从后排即可输出战士。
- **战士侧维持单战士 + 单体集火**(不动),AoE 不做。
- **复用现有结算管线**:`enemies: Array[Entity]`、`select_target` 集火、`enemy_defeated`/掉落/清场计数 全沿用,只在
  "敌方怎么组一波"(SceneConfig 多敌/站位)与"谁够得着战士"(in_range 门控)两处点亮。
- **明确不含(推后)**:战士 / 任何玩家角色的 **AoE / 群伤**(→ 未来技能 / 法师,F-AOE);**真 2D 走位 / 抛射物 / 碰撞**
  (违 #7,永不做);**多波编排的复杂 boss 团**(v1 杂兵波即可);**敌人互相增益 / 阵型 buff / 召唤**;
  **战士主动够后排的机制**(嘲讽 / 穿透 / 跳后排——都推后)。

- **🟡 F-NUM(交 num-smith)** — 团战数值全套:**一波规模**(2–4 的确切上限 / 是否随场景深度递增)、**近/远配比**、

- **🟢 F-AOE(明确推后,记录)** — 用户拍板 v1 **战士单体、AoE 留技能**。团战 §3 的"远程够不到"张力**正是 AoE 的设计留白**:
  未来给法师 / 战士技能加群伤时,这层站位结构(前排墙 + 后排远程)会**天然成为 AoE 的用武之地**。v1 不抽象、不预埋
  AoE 接口(守"不为没影的系统提前抽象"),只在设计上留好这个未来挂点。
- **🟢 支柱 1 相容(已判定相容)** — 团战增加的是**画面密度与压力**,不增加操作:仍全自动集火、玩家不点不拖。
  唯一风险是"一群"在 800×250 窄条里**挤不挤得下 / 看不看得清** → 留 playtest + 统一 UI 轮处理(站位呈现是表现层)。
- **🟢 Producer 知会** — 08 scope 锁在"**敌方一波多敌 + 近/远站位门控**",战士侧零改动、AoE 推后。若后续想把"多波编排 /
  boss 团 / 玩家 AoE"纳入,属新功能,应另开条目而非塞进 08。
