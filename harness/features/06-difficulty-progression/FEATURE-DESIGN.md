# FEATURE-DESIGN — 06 难度推进:撞墙 → 变强 → 冲过(v1 闭环收口)

> 本功能是 v1 垂直切片的**收口件**:把前面所有功能(挂机打怪 02 / 掉装看得见 04 /
> 城镇换装·强化 05 / 团战 08)串成一个**闭合的回报循环**——"刷→调→再挑战"。它**不造新系统**,
> 而是把地基里已有的进度骨架(`ProgressionController` 的团灭回退 / 卡关 GRINDING / 推进 /
> Boss 永久解锁)点亮成一个**玩家能感受到的"墙与突破"**。体感取向已与人拍板:**混合·软底+主动冲**。

## 1. The fantasy(玩家该感受到什么)

**"我撞到一堵打不过的墙——但我不慌;我去把自己变强,回来,这次冲过去了。那一下'我变强了、所以够到了'的踏实,就是我玩到现在的回报。"**

- 撞墙**不焦虑、不惩罚**:队伍自动退回安全区继续刷,进度一格不丢、装备照常掉(支柱 1:它是陪伴,不是任务)。
- 卡关**一眼看懂**:余光扫一下就知道"卡在这了,不是卡 bug",而不是盯着屏幕猜发生了什么(支柱 1:瞥一眼看懂)。
- 变强**有的放矢**:墙给了"为什么要回城"的理由——刷到的更好装备 / 攒的材料强化,**就是为了够到这堵墙后面的东西**(支柱 2:变强是为了够到下一个够不着的怪)。
- 突破**是高光**:过墙 / 解锁新关那一下,是整个 v1 体验的兑现落点,要比普通过场更有分量,让人想立刻去够下一堵墙。

这是把支柱 2 的"刷→调→再挑战"闭环**真正闭上**的一环:前面的功能造了"刷"和"调",06 造的是**"再挑战"那一下值不值**。

## 2. Core loop(动作 → 结果 → 反馈 → 再来一次的动机)

```
挂机自动打 (PROGRESSING)
   │  推进到一档当前打不过的内容
   ▼
撞墙:团灭 → 自动软退到最近能打的场景,继续安全刷 (GRINDING)
   │  ← 进度不丢、装备照掉(软底:陪伴不打断)
   ▼
卡关可读:余光看懂"卡在这了" + 轻量"去城里变强或许能过"的邀请(不弹窗打断)
   │
   ▼
玩家主动:回城(05 已有)换装 / 强化 → 变强
   │
   ▼
玩家主动:点"推进"再战那堵墙(主动冲:这是"我觉得我行了"的表达)
   │
   ├─ 还不够 → 再软退安全刷(回到 GRINDING,零惩罚,再攒再调)
   │
   └─ 够了 → 冲过墙!Boss 永久解锁 / 推进下一关 → 高光庆祝
                 │
                 ▼
            下一堵墙(回到顶部,但你更强了)
```

**循环为什么闭合**:每堵墙后面是"更深内容 → 更高 ilvl 掉落(BALANCE i6 单调闸门)→ 能再变强 → 够到再下一堵墙"。
突破的兑现感 = 玩家主动投入(回城调整 + 主动推进)换来的可见进步,**这就是再来一次的动机**。

## 3. Rules & states(具体规则、胜负、边界态)

### 3.1 复用的进度骨架(已在地基,06 不重造)
- `ProgressionController` 已有四态:`PROGRESSING`(正常推进)/ `GRINDING`(卡关安全刷)/
  `STAGE_CLEAR_COUNTDOWN`(通关倒计时)/ `RESTING`(修整 stub)。
- **团灭 → `retreat_after_wipe()`**:自动退回更早的安全场景、进 `GRINDING`、复活、把"刚才那档"记为推进目标(`advance_target`)。**Boss 永久解锁绝不回退**(已保证)。
- **推进 → `request_push()`**:GRINDING 态下入队,本轮结束跳回 `advance_target`(= 再战那堵墙)。
- 06 **不新增**进度 FSM 状态;它定义的是"哪一档算墙"、"卡关怎么对玩家可读"、"突破怎么庆祝"。

### 3.2 "墙"是什么(用现有内容,不造新关)
- 墙 = **当前装备打不过、但回城变强后能打过的那一档更硬内容**。用现有关1 / 关2,**不做难度曲线机器、不加新区域 / 新 Boss 机制**(守 Producer 的 LESS)。
- 墙落在**关2 入口 或 关2 Boss**(具体哪一档、需多大变强增量才过 = 数值命门,交 num-smith,见 Flags)。
- **核心不变量(交 num-smith 兑现)**:墙要满足"**关1 顶配 gear 纯 AFK 过不去、回城用关2 级掉落/强化变强能过**"——太松则没有墙(闭环空转)、太硬则违支柱 1(逼玩家硬磨)。

### 3.3 胜 / 负 / 边界
- **胜(过墙)**:清掉被 gate 的更硬内容 → Boss 永久解锁、推进下一关 → 触发突破高光。
- **"负"在 idle 里不是失败态**:团灭 = 自动软退安全刷,**不掉永久进度、不掉装备**(软底)。这是有意的:支柱 1 不容许"惩罚式失败"。
- **边界态**:
  1. **一直不回城、也不点推进** → 永远在 GRINDING 安全刷,零惩罚的纯陪伴态(可接受)。**但**:即便纯 AFK,掉落仍会被动累积(空槽自动填,i3)→ 墙**最终**也会变得可过,只是远慢于主动回城调整。墙是"软"的:绝不死锁玩家,只是奖励主动参与(支柱 2)。
  2. **回城变强后出城不回血**(不变量 #11 / i5,05 已定)→ 残血再战墙是否公平,由 num-smith 在定墙时一并校。
  3. **变强了但还差一点** → 再撞墙 → 再软退 → 再攒再调,循环不卡死、不报错。
  4. **打通最后一关 Boss** = v1 内容走到终点 → 给一个"打通当前内容"的收尾反馈(非新系统,见 §4),之后可继续在终点关安全刷陪伴。

## 4. Feedback & juice intent(关键事件玩家该看到/听到什么 —— 只给意图,不指定素材)

> 交给 Art / Game Feel,本节只定"该传达什么感觉"。占位程序美术先满足可读,正式皮 / 音 / 特效并入全局 UI·juice 统一轮。

- **撞墙那一刻**:安静、非报错式的状态变化——传达"在这磨着呢",**不是**红字 error / 不是焦虑弹窗(守支柱 1:它是陪伴)。
- **卡关可读**:余光能看懂"当前卡在哪一档"(进度指示 + 一个克制的"卡关中"状态),无需盯屏或读日志。
- **回城邀请**:轻量、不打断的提示"去城里变强或许能过"——**绝不弹窗 / 绝不强制**(支柱 1)。玩家可以无视它继续陪伴。
- **点推进再战**:玩家的主动动作要有明确回应("发起再挑战 / 冲锋"的反馈),让"我觉得我行了"这一下有重量。
- **突破墙 / 解锁新关**:**这是 v1 的高光**,要比普通过场更有分量(里程碑级庆祝)——标记"这一下必须爽"。具体特效留 juice 轮,但意图优先级最高。
- **进度可读**:打到哪关 / 哪场、卡在哪、解锁到哪,余光可读(支柱 1)。

## 5. Success criteria(怎么判断好玩 / 成立 —— playtest 问题)

1. **可读**:撞墙时我是否一眼看懂"卡在这了、不是 bug",而非盯屏猜?
2. **软底安心**:软退安全刷时我是否安心(没掉进度、还在出装),而非焦虑/被惩罚感?
3. **闭环兑现(核心)**:回城变强 → 回来点推进 → 过墙,这一下是否有"**我变强了所以够到了**"的踏实兑现感?(这是 06 成败的命门)
4. **墙的公平性**:关1 顶配 AFK 是否过不去、回城用关2 级掉落/强化变强是否能过?(num-smith 校,但 playtest 主观体感是最终裁判)
5. **守支柱 1**:整个"卡关 → 回城 → 推进"过程是否都能"瞥一眼 + 点一下"完成,从不逼我盯屏 / 搓实时操作?
6. **突破分量**:过墙的庆祝是否够重,让我想立刻去够下一堵墙?
7. **不死锁**:即便我懒得回城,陪伴态是否依然舒服、且不会把我永远焊死在墙前(被动累积最终也能过)?

## 6. Minimal version(最小可交付,但仍兑现幻想)

**核心 = 一堵真墙 + 卡关可读 + 突破高光,全用现有骨架与内容。**

- **用现有关1→关2 两关、现有 GRINDING + 团灭回退 + 推进骨架**,不造新内容、不做难度曲线机器、不加动态难度。
- **一堵真墙**:num-smith 把"关2 入口(或关2 Boss)"调成"关1 顶配 AFK 过不去、回城变强能过"(§3.2 不变量)。这是最小版的命门,**必须有**。
- **两处反馈意图落地**(占位程序美术即可):① 卡关可读(状态 + 进度指示)② 突破庆祝(比普通过场重)。其余 juice 留统一轮。
- **玩家动作仅复用现有两个**:回城(05)+ 点推进(现有宏操作按钮)。**不加新 UI 系统**。

**可砍 / 留后(仍兑现幻想)**:
- 多堵墙的曲线编排、动态难度、新 Boss 机制、关3+ 新区域 → v2(守 LESS)。
- 关2 团战铺多敌波(08 deferred):**墙体验不依赖它**——单敌更硬也能成墙;铺波属"更丰富"非"必须"(见 Flags)。
- "自动再战墙(变强后自动重试)":考虑过,但**混合取向明确要玩家主动冲**,且自动检测会加逻辑——v1 = 手动推进即玩家能动性,自动重试留后续可选。
- 正式皮 / 音 / 突破特效 → 全局 UI·juice 统一轮。
