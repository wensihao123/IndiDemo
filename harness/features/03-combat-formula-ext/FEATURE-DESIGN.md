# FEATURE-DESIGN — 扩 02 战斗公式以承载装备维度 (Combat Formula Extension)

> 本功能源自 Producer/用户裁定(2026-06-18):04 装备的基底+词缀要表达攻速/护甲/闪避/暴击/回血等维度,
> 而 02 现模型只有"每 tick 全队 `attack` 之和扣血、敌人每 tick 回打前排平伤",撑不起。故先扩战斗公式,
> 作 04 的地基。**这是一次对 02 战斗解算的受控扩展(reopen),不是新玩法**——它本身不直接面向玩家,
> 但它决定了玩家日后"换装→打法变样"的全部质感。**战斗仍 100% 自动(支柱 1)。**
>
> ⚠ 数值约定(用户拍板):本设计只定**公式形态与语义**;一切具体数值(攻速基准、护甲 K、暴击率/倍率、
> 闪避率、回血速率、狂暴阈值/曲线)先用"大致合理"占位跑通,**不设硬上限**,精调统一留 §7 的数值设计专章。

## 1. The fantasy / 玩家体验
"我的装备改变的是战士**怎么打**,而不只是一个更大的数字。" 一把快攻暴击匕首,出手如雨、时不时一道暴击尖峰;
一身重甲+回血,稳稳扛住、磨死对手;一件高闪饰品,赌它一串不挨打。玩家**一眼能看出不同 build 的战斗气质不同**
——这正是支柱 2"主动构筑"的手感来源。而无论怎么配,他都不用操作:配好了,挂着看战士按自己的节奏卖命。

补一层系统幻想:**没有"互相打不动"的尴尬僵局**。打得过就利落收掉,打不过——敌人会**狂暴**起来把你推平,
干脆利落地告诉你"这怪你现在啃不动,去变强"。绝不会卡在一只怪身上无限耗着,让人怀疑游戏卡死了。

## 2. Core loop / 核心循环
本功能是**战斗解算层**的循环(喂给 04 的物品层、最终汇入支柱 2 大闭环):

**单场结算(自动,每 0.1s 一 tick):**
1. 每个存活成员按各自**攻速**累计出手进度 → 到点打出**一次离散命中**:基础伤害=该成员 `attack`,
   每次命中 roll **暴击**(中→×暴击倍率)。
2. 敌人按自己的**攻速** cadence 出手,打最前存活成员 → 该成员先 roll **闪避**(中→本次伤害归 0),
   未闪则按**护甲**做减伤。
3. 每 tick 给存在的成员按**每秒回血**恢复(场内即时,封顶满血)。
4. **软狂暴**:同一只敌人被缠斗超过时间阈值后,其输出按曲线**陡增**,直至本场分出胜负。
5. 结算:敌人血≤0 → 击败(沿用 02 的击杀→掉落→推进);全员倒下 → 团灭(沿用 02 的回退/无尽刷)。

**宏观(跨功能,为什么要扩这些维度):** 扩出的维度 → 04 让装备基底/词缀去填它们 → 玩家换装,战士的
出手频率/尖峰/减伤/续航**真的变样** → 够到旧属性profile够不着的关 → 回去刷更好的装。

## 3. Rules & states / 规则与状态

### 3.1 出手 cadence(模型 a:离散命中)
- 每个 `PartyMember` 与 `EnemyDef` 拥有 **attack_speed(每秒出手次数)**。
- 每 tick:`progress += attack_speed × tick_seconds`;当 `progress >= 1.0` 则打出一次命中并 `progress -= 1.0`
  (单 tick 内可触发多次命中,循环扣减;留防卡死的单 tick 上限)。
- 一次"命中"是离散事件——**暴击/(未来吸血等)按"每次命中"结算**,这是攻速能与它们协同的根因。
- ⚠ 这把 02 的"每 tick 全队 `attack` 之和"换成"按 cadence 的离散命中",**DPS 基准被重定**(原 = 攻击×10/秒)。

### 3.2 进攻:暴击
- 每次命中 roll:`if rng < crit_chance → 伤害 ×= crit_mult`。
- RNG 落在进攻侧——背一发暴击只是杀慢一点,不致命,契合支柱 3"惊喜不焦虑"。

### 3.3 防御:闪避 → 护甲(顺序固定)
- 被命中时**先 roll 闪避**:`if rng < dodge_chance → 本次伤害 = 0`(全有/全无)。
- 未闪则**护甲减伤**:`减伤率 = armor / (armor + K)`(K = 配置常数),`实伤 = 入伤 ×(1 − 减伤率)`。
  递减曲线,**永不到 100%**,避免"堆够护甲=无敌"的破点。
- 闪避(赌、尖峰)与护甲(稳、平滑)质感不同质,故并存;两者都"延长生存",冗余度交 playtest 看(§7)。

### 3.4 恢复:每秒回血
- 每 tick:`current_hp = min(max_hp, current_hp + hp_regen × tick_seconds)`。
- **只在场内生效**;02 既有的"过场景/通关/满轮全队回满、无跨场景损耗"**不变**。
- 无硬上限(用户定);防"回血≥入伤→永不死"靠 **3.5 软狂暴**兜底,而非靠 cap。

### 3.5 软狂暴(anti-stalemate,本功能新增)
- 每**一只敌人**的战斗带一个缠斗计时器(sim 时间);超过 **enrage_threshold** 后,敌人输出按曲线**递增**
  (狂暴系数随时间继续涨),直到本场分出胜负;**换敌人后计时器重置**。
- 作用:**保证每场战斗必然终止**——杀得动就在阈值前收掉(不触发狂暴);杀不动=你欠强,狂暴会把你推平
  → 团灭回退去变强(支柱 2)。根除"双方互相打不动→卡在一只怪上无限耗"的死局。
- 自洽性:回退去刷的较弱场景,你能在阈值内轻松清,**狂暴不会误伤正常farming**;它只咬"对你太肉的怪"。
- 状态:每场有 `NORMAL → ENRAGED` 两态(过阈值后进 ENRAGED,本场结束复位)。

### 3.6 数值来源(本功能阶段,尚无装备)
- 本功能**不含装备物品**(那是 04)。新维度的值来自:战士走 `CombatDirector` 的 `@export` 配置 +
  `EnemyDef` 的 Resource 字段。**默认值要让"无装备战士"的行为≈现状的合理重定**(见 §6 占位值)。
- 日后 04 把"基底+词缀"汇总后**写进 `PartyMember` 的这些字段**——本功能只负责"字段存在且被战斗正确消费"。

### 3.7 部位↔基底映射(对齐 04,本功能只实现"公式维度"不实现物品)
- 武器基底 = **攻击 + 攻速**;护甲基底 = **护甲**;饰品基底 = **{生命 / 闪避 / 回血} 三选一**(灵活位,暂定,§7)。
- **暴击**及其余维度作为**词缀**挂载(饰品尤其适合扛暴击词缀)。

### 3.8 对 02 的侵入边界(务必最小化,reopen 不重写)
- **会动:** `tick_combat` 的伤害施加(改为 cadence 离散命中 + 暴击/闪避/护甲/回血结算 + 狂暴系数);
  `PartyMember` 加字段(attack_speed / armor / dodge_chance / crit_chance / crit_mult / hp_regen);
  `EnemyDef` 加字段(attack_speed;狂暴阈值/曲线可放 EnemyDef 或全局配置,留 Planner)。
- **绝不动(扩展非重写):** `loot_dropped` 事件流与 roll 逻辑、`boss_cleared`/永久解锁、团灭回退/无尽刷状态机、
  5s 倒计时、后台持续模拟(tick 驱动)、4 格队伍地基、"过场景/通关/满轮全队回满"的回血粒度。
- **胜负判定不变:** 敌血≤0=击败、全员倒下=团灭——下游(掉落/推进/回退)只认这两个事件,不受公式改动影响。

## 4. Feedback & juice intent / 反馈意图(交 Art/Audio/Game Feel;守 02 符号/轻演出版天花板)
- **暴击命中:** 伤害飘字更大 + 暴击色(让"尖峰"看得见,支柱 3 的小确幸落在进攻侧)。
- **闪避:** 被攻击时冒一个 "闪避/MISS" 字样(否则玩家看不懂"为什么这下没掉血")。
- **护甲减伤:** 通常无需专门演出(数字本身变小即可),避免噪音。
- **回血:** 血条上一缕柔和的回升提示(余光可辨,不抢注意力)。
- **⚠ 软狂暴必须可读:** 进 ENRAGED 给一个明确但克制的"敌人狂暴!"提示(图标/染色/一行日志)。
  这是支柱 1 的关键——否则战士"突然暴毙"会让人莫名其妙;有了它,玩家一眼懂"哦它太肉了,该去变强"。
- 所有以上是**意图非素材**;具体表现归 Art/GameFeel,且都在符号/轻演出版范围内(横版全套演出仍在 Later)。

## 5. Success criteria / 成败判断(playtest 问题)
- 不同 build 的**战斗气质是否真的可辨**?(快攻暴击 vs 重甲回血 vs 高闪)——这是本功能存在的意义。
- 暴击在进攻侧是"爽"还是"无所谓"?是否带来"哦一发大的!"而**不**制造焦虑?
- **护甲/闪避/回血三条防御轴是否各有存在感,还是糊成一坨**"反正就是更耐打"?(若严重冗余,§7 再砍并)
- **软狂暴**:是否成功消灭了"卡在一只怪上耗着"的死局?触发时玩家是觉得"合理,该去变强"还是"被廉价计时器坑"?
  阈值体感是否过短(没机会翻盘)/过长(干耗很久才狂暴)?
- 支柱 1 可读性:战士的"暴毙"(连串没闪/被狂暴)玩家**能不能一眼看懂死因**?
- 闭环是否仍被门槛把住——**续航再强也不能无限苟**(靠狂暴),"够不着→去变强"是否还成立(支柱 2)?

## 6. Minimal version / 最小版本(仍交付幻想:维度可辨 + 战斗必然终止)
- **cadence 离散命中模型**(attack_speed 驱动出手)——这是支点,不可省。
- 六个新维度全部接入并被战斗正确消费:`PartyMember` 的 attack_speed / armor / dodge_chance /
  crit_chance / crit_mult / hp_regen;`EnemyDef` 的 attack_speed。
- **软狂暴**(阈值 + 递增曲线 + ENRAGED 态 + 可读提示的钩子)。
- 各维度先用**"大致合理"占位值**跑通(不精调、不设 cap),例如(全部暂定,留 §7 专章):
  - 战士默认:attack≈6、attack_speed≈1.0/s、armor=0、dodge=0、crit_chance=0、crit_mult≈2.0、hp_regen=0
    (即"裸装战士"≈把现 60DPS 重定到 6×1.0,**02 怪血需随之等比重调**)。
  - 敌人:attack_speed≈1.0/s;软狂暴阈值≈20–30s、之后输出按温和指数/线性递增(形状留专章)。
- **02 回归收口**:重跑 02 全套 gdUnit4 + headless 校验;DPS 重定会改变击杀/团灭的 tick 时序,
  **时序敏感的断言需更新**(行为结构不变:击杀→推进、团灭→回退)。
- 暂不做:装备物品本身(04)、吸血(本轮未选,留 v2)、敌人侧的暴击/闪避/护甲(留作日后调怪旋钮)、
  套装/宝石/重铸、等级经验、团战。

## 7. Flags / Open questions
- **F1〔数值设计专章 · 必做后续〕** 本设计只定公式形态,**全部数值留专章精调且暂不设硬上限**:攻速基准与
  上限行为、护甲 K 与曲线、暴击率/倍率、闪避率、回血速率、**软狂暴 enrage_threshold + 递增曲线形状**、
  以及 02 怪血/战士的等比重定。建议数值专章单独立项或作本功能的二阶段;落地前先用 §6 占位值验证机制跑通。
- **F2〔饰品基底三选一〕** 饰品基底 ∈ {生命 / 闪避 / 回血} 暂定未定死;饰品定位为"灵活词缀位",
  基底取哪个交后续(可能等 04 设计物品时一并定)。本功能只需保证这三维度公式都在,不阻塞。
- **F3〔防御冗余观察〕** 护甲/闪避/回血三条都"延长生存",playtest 若发现辨识度糊,保留"砍并其一"的余地
  (设计上已让三者机制不同质:稳减伤/赌规避/持续恢复)。
- **F4〔回归风险,交 Planner/Implementer〕** reopen 02:cadence 改写 `tick_combat` + 加字段,**必须**重跑
  02 测试与 headless;时序敏感断言更新;确认掉落/解锁/回退/后台模拟/回血粒度未被破坏(§3.8 边界)。
- **F5〔软狂暴归属与配置位〕** enrage 阈值/曲线放 `EnemyDef`(每怪可调)还是全局配置(统一)?交 Planner 定
  (倾向:全局默认 + EnemyDef 可选覆盖,守"走配置不硬编码")。狂暴是否对 Boss / 无尽刷场景一视同仁?
  默认一视同仁(farming 场景因够弱不会触发);留 playtest 复核 §3.5 自洽性。
- **F6〔范围提醒〕** 本功能**只扩"装备要用到的战斗维度"**;等级/经验、团战(单场多敌)仍在 BACKLOG/Later,
  **不顺手并入**(守 hard-NO)。吸血本轮未选,如日后要,与回血同 niche,二选一即可。
- **F7〔可读性硬要求〕** 软狂暴与闪避的"死因可读"是支柱 1 验收点(§4/§5),非可选润色——Art/GameFeel 须落地提示钩子。
