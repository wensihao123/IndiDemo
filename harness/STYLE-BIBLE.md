# STYLE-BIBLE(项目专属画风)

> 它**叠在** `style-basic-2d.md`(工程纪律基线)**之上,不取代它**。本文件只回答"这游戏长什么样"
> (调色板/线条/视角/母题);"素材怎么造/导入/进场景树"一律以 `style-basic-2d.md` 为准。
> 现处于**占位阶段**:很多项留 TODO,等有正式美术再充实。锁死的是大方向,不是细节。

## 0. 基调(一句话)
类 **MapleStory** 的**手绘 2D**:圆润可爱、暖调、清晰描边、轻盈治愈。**非像素风**——
服务支柱 1"它首先是个伙伴"(看一眼就安心、不冰冷)。

## 1. 渲染风格(已锁)
- **平滑风,不是像素风** → `texture_filter = Linear`;导入开 `Fix Alpha Border`(〔EI〕执行)。
- 禁止把本作做成硬像素/点阵观感。

## 2. 分辨率基准(已锁)
- **主游戏区基准画布 = 800×250**(居中);美术以此为逻辑参照。
- 窗口为**全屏宽**,主区两侧由**占位背景**延展铺满,适配不同屏宽(见 project-context §1)。

## 3. 线条与形状(已锁,2026-06-18)
- **描边**:统一描边色 **`#3a2e2a`(暖棕,非纯黑)**;主轮廓在素材原始画布上约 **3px**,
  内部分线约 **2px**(随素材缩放,保持"看得出描边但不抢眼"的中等粗细)。
- **转角**:圆润、半径偏大,避免尖锐/写实硬角。
- **体型**:偏 Q,头身比约 **1:1.3~1:1.5**(头大、亲和)。
- **明暗**:平涂为主 + **单层柔和 cel 阴影**(取该色块主色压暗约一档,≈ ×0.85),
  高光极简或省略;整体**明度偏高、低饱和**。**禁止**写实渐变/厚涂、强噪点。

## 4. 调色板(已锁 hex,2026-06-18 —— 方案 A「暖木治愈」)
> 占位阶段结束,以下为项目正式 palette。新素材一律落在这套色里;偏色即不通过验收。

| 角色 | 名称 | hex | 用途 |
|------|------|-----|------|
| 描边 | 暖棕 line | `#3a2e2a` | 所有素材统一描边色(非纯黑) |
| 主色 | 暖木 | `#c98a5e` | 城镇 / 暖调主体 |
| 辅色 | 奶白 | `#f5e9d6` | 高明度底 / 留白 / 浅面 |
| 探索 | 清新绿 | `#7fb069` | 户外探索氛围主绿 |
| 阴影 | 深绿 | `#4a7c59` | 绿系暗面 / 远景 |
| 强调 | 暖橙 | `#e8943c` | 点睛 / 关键反馈暖色 |
| UI 底 | 深紫灰 | `#2b2536` | HUD / 面板底(衬托高明度前景) |
| UI 文字 | 奶白 | `#f2e9dc` | UI 文本 / 图标线 |

**稀有度色(与现有代码一致,勿改):** 白 `#d9d9d9` · 蓝 `#6699ff` · 金 `#ffd24a`。
> 这三色已硬用于掉落 FX(`combat_view.gd` `RARITY_COLOR`);改 hex 会脱节,锁死。

**UI 反馈专用色(2026-06-20 新增 —— 仅用于运行时 juice / 数值显形,*不用于素材填色*):**
| 角色 | 名称 | hex | 用途 |
|------|------|-----|------|
| 反馈·增益 | 反馈绿 | `#7fb069` | 复用清新绿:数值↑、换装/自动填空成功闪、确认 |
| 反馈·损失 | 反馈红 | `#d6604a` | 暖红(新增):数值↓、扣血、失败/拒绝提示 |
| 反馈·关键 | 强调暖橙 | `#e8943c` | 复用强调色:强化成功闪、末关里程碑庆祝 |
> 反馈红是**唯一新增 hex**,且**限定 UI 反馈用途**——不进角色/背景/图标的固有填色,避免污染「暖木治愈」主 palette。

## 5. 视角(已锁,2026-06-18)
- **横版正侧视(side-view)**,类 MapleStory;无强透视。
- 所有战场实体(战士、敌人)**脚底站在同一条地平线**上(共享 horizon)。
- **不做**地面厚度/俯角;脚底阴影 = **运行时叠一枚柔和椭圆**(不烘焙进角色底图,遵
  style-basic-2d §3),保证朝向 `flip_h` 与状态染色不连带阴影出错。

## 6. 动画约定(已锁基线)
- **首版一律"运行时假动画"**:`AnimationPlayer` 动 `position/scale/modulate` 做待机微动
  (呼吸/上下浮),**不画帧序列、不把动作烘焙进 PNG**(遵 style-basic-2d §3、§6.5)。
- 帧动画(若日后需要)状态顺序:`idle → move → action → hit → death`,先少后多。

## 7. Do / Don't(占位阶段红线)
- DON'T:把血条/数字/选中框/状态染色烘焙进底图(全运行时叠加,见 style-basic-2d §3)。
- DON'T:混入像素风;文字进图标 PNG。
- DO:透明 PNG、脚底锚点、`flip_h` 复用朝向、命名全小写 snake_case。

## 8. 界面系统 / UI System(已锁,2026-06-20 —— 全局 UI 轮地基)
> 这一节统一所有功能(05 城镇 / 04 掉落面板 / 02 战斗 HUD / 06 占位呈现 / 08 团战)的界面外观。
> 落地手段(`Theme.tres` / `StyleBox` / `NinePatchRect` 节点)由 **EI** 建,本节只定**风格值**;
> 文字一律 `Label`/`RichTextLabel`、可拉伸件一律 `NinePatchRect`,**绝不烘焙文字进图**(style-basic-2d §6.4)。

### 8.1 排版纪律(主区 800×250,余光可读 = 支柱 1)
- **间距阶梯(spacing scale):** `4 / 8 / 16` px —— 紧凑间距 4、标准内边距 8、区块外边距 16。统一取这三档,勿散值。
- **字号阶梯(type scale):** 标题 **16** / 正文 **12** / 小字(次要·数值差)**10** / 强调大字(里程碑·飘字)**20**。(沿用现有正文 12 基线。)
- **圆角:** 面板 **8px** / 按钮·输入 **6px** / 小标签 **4px**(呼应 §3 圆润、半径偏大)。
- **遮挡红线:** 战斗 HUD 不遮战场中心(支柱 1 余光可读);城镇面板(暂停态)可占较大区,但保留关键信息一眼可读。

### 8.2 面板(`StyleBoxFlat` / `NinePatchRect`)
- **面板底:** bg 深紫灰 `#2b2536` @ **~92% alpha**(悬浮窗半透,衬高明度前景)、圆角 8、边框 **2px 暖木 `#c98a5e`**、内边距 8。
- **分隔线:** 1px 暖棕 `#3a2e2a` @ ~40% alpha。
- 若用 `NinePatchRect` 底图:`patch_margin ≥ 圆角+边框`(≈10px),九宫格切图规格留 per-feature ASSET-SPEC。

### 8.3 按钮(`Theme` 四态 `StyleBox`)
- **normal:** bg 暖木 `#c98a5e`、文字 奶白 `#f2e9dc`、圆角 6、边框 2px `#3a2e2a`。
- **hover:** bg 提亮 ×1.10(或叠 暖橙 `#e8943c` ~10%)。
- **pressed:** bg 压暗 ×0.85 + 内容下移 1px。
- **disabled:** 去饱和 + 50% alpha,文字 60% alpha。
- **focus:** 2px 暖橙 `#e8943c` 外描边。
- 文字走 `Label`/`Theme`,**绝不烘焙进按钮图**。

### 8.4 图标语言(`assets/ui/icons/`)
- **基准画布 32×32**(游戏内显示 16–24),透明 PNG。
- 统一暖棕描边 `#3a2e2a`(主轮廓 ≈2px@32 画布)、内部平涂取主 palette 色、圆润强轮廓高对比。
- **一图一概念**;冷却 / 数量 / 等级 / 禁用态全运行时叠(`Label`/`TextureProgressBar`/`modulate`),**不出多份带数字的图**。
- 命名 `icon_<concept>.png`(全小写 snake_case)。

### 8.5 数值 / 文本显形
- 8 维属性 & 对比面板差值:正文 12,轴名 奶白;差值用 **反馈绿↑ / 反馈红↓** + 箭头字符(§4 UI 反馈色)。
- 稀有度文字/边框色 = 掉落 FX 三色(白 `#d9d9d9` / 蓝 `#6699ff` / 金 `#ffd24a`),名称与边框一致。

### 8.6 框架件:九宫格皮(2026-06-20 —— 全局 UI·juice 轮,据用户参考图)
> **§8.2 的 `StyleBoxFlat` 数值(底色/圆角/边框/内距/间距/字号阶梯)仍是排版与内距的事实源**;本节只把
> "框的皮"从纯色 flat **升级为暖木 + 金花边的九宫格(NinePatch)框架件**,以贴近用户参考图的 MapleStory
> 暖木公告板观感。落地手段(`Theme.tres` + `StyleBoxTexture`/`NinePatchRect`)归 EI,本节定**风格取向**,
> 具体切边/尺寸/命名落 per-feature ASSET-SPEC(首份 = `features/10-ingame-flow-nav/ASSET-SPEC.md`)。
- **取向**:框 = 半透深紫灰 `#2b2536@~92%` 内底 + 暖木 `#c98a5e` 边 + 四角金/暖橙 `#e8943c` 花头;
  **克制**——花边不抢注意、不压低文字对比(守支柱 1 余光可读)。九宫格件**四角可花、四边须平铺无缝**。
- **复用纪律**:出**一套小而通用的框架件**(板框/内嵌框/标题牌/按钮底/页签/槽框/列表行/头像框),
  **跨板块复用**,不为每板块单独画框。`_sel` 选中态用暖橙;稀有度边框/禁用/hover/pressed/focus 全运行时叠(§8.3)。
- **退路**:若某项要更省/更冷,可退回纯 `StyleBoxFlat`(§8.2 数值)不出框贴图——两者数值口径一致,可无缝降级。

## 9. juice / 反馈约定(已锁,2026-06-20)
> **全部运行时叠加、零烘焙**(`modulate`+`Tween` / `AnimationPlayer` / 粒子,style-basic-2d §3);落地由 EI。
> **红线:守支柱 1(冷静、不抢注意)+ 支柱 3(非赌场)** —— 反馈**短促 0.2–0.4s、不阻塞、不连续闪烁、不做抽奖式演出**;同屏密集时透明度/粒子量可控。

| 事件 | 反馈 | 色 | 时长 |
|------|------|----|------|
| 自动填空 / 换装成功 | 该件 `modulate` 闪后 `Tween` 回 | 反馈绿 `#7fb069` | ~0.3s |
| 装备强化成功 | 暖橙闪 + 上浮 "+L" `Label` 淡出 | 强调暖橙 `#e8943c` | ~0.5s |
| 对比面板差值 | 即时显形(静态) | ↑反馈绿 / ↓反馈红 | — |
| 掉落迸出 | 按稀有度色迸出(已在 `combat_view` `RARITY_COLOR`) | 白/蓝/金 | 短 |
| 普通过关 | 较轻一档全屏闪(与里程碑区分) | 强调暖橙(低 alpha) | ~0.3s |
| 末关里程碑庆祝(06) | **更重**的全屏闪 + 日志 | 强调暖橙(高 alpha) | ~0.6s |
| 卡关可读(06 GRINDING) | 进度读出 + 回城邀请行(静态文本) | 正文/小字色 | — |
| 受击 | 目标闪白(`modulate`) | 白 | ~0.1s |

> 伤害飘字 / 横版攻击-受击帧演出属 **02 战斗演出**(本轮"中档"不含帧序列,见 §10);若做轻飘字,沿用本节 juice 红线。
